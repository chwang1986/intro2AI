07 多人协作

模拟两人协作：开发、推送、拉取、合并冲突

场景两个人怎么一起工作

实际工作中，一个项目通常由多个人共同开发。每个人在本地开发自己的功能，然后把代码推送到远程仓库。Git 会自动合并不冲突的修改，如果两个人改了同一处代码，就需要手动解决冲突。

这个页面模拟了两个人（用户A 和 用户B）共同开发一个项目的完整流程。点击下方「创建协作环境」按钮，系统会自动生成远程仓库和两个本地克隆。

要点协作的核心流程

1. 各自开发：每个人在自己的分支上开发，互不干扰。开发完成后 push 到远程。

2. 获取更新：在开始工作前，先 git pull 获取队友的最新代码，避免在过时的代码基础上开发。

3. 解决冲突：如果两个人的修改重叠了，Git 会标记冲突位置。需要手动编辑文件，保留正确的内容，然后重新 add 和 commit。

4. 代码审查：通过 Pull Request（GitHub）或 Merge Request（GitLab）让队友审查代码，确认无误后再合并到主分支。
