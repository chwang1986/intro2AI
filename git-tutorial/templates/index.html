讲解Git 是什么

想象你在写一份重要的文档。你改了又改，保存了很多个版本：「报告_v1.doc」「报告_v2.doc」「报告_最终版.doc」「报告_最终版_真的.doc」。最后你也不知道哪个才是真的最终版。

Git 就是来解决这个问题的。它帮你记录每一次修改，你可以随时回到任意一个历史版本，也可以对比两个版本之间的差异。更重要的是，它让你可以放心地尝试新想法——如果改坏了，随时可以回退。

Git 的核心概念只有三个：工作区（你正在编辑的文件）、暂存区（准备提交的文件快照）、仓库（所有历史版本的记录）。理解这三者的关系，就理解了一半的 Git。

实例初始化你的第一个仓库

打开终端，进入一个你想要管理的项目文件夹，运行下面的命令。这些命令会创建一个隐藏的 .git 目录，Git 的所有记录都保存在这里。

动手操作
# 进入项目文件夹
cd my-project

# 初始化 Git 仓库
git init

# 查看仓库状态
git status

运行 git status 后，你会看到当前分支是 main，还没有提交记录。

代码实现
# 创建一个新文件
echo "Hello Git" > README.md

# 把文件加入暂存区
git add README.md

# 提交到本地仓库（-m 后面是提交说明）
git commit -m "初次提交：添加 README"

# 查看提交历史
git log
