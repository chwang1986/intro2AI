08 综合实战

完整工作流：新建 → 修改 → 提交 → 分支 → 合并 → 推送

场景一个完整的开发流程

假设你接到了一个新任务：给项目添加用户登录功能。下面是使用 Git 的完整工作流，从接到任务到代码上线。

流程8 步标准工作流

1. 更新代码：git pull origin main —— 确保本地代码是最新的。

2. 创建分支：git branch feature-login && git switch feature-login —— 在独立分支上开发，不影响主分支。

3. 开发功能：修改代码、创建文件、测试功能。

4. 提交更改：git add . && git commit -m "feat: 添加用户登录功能" —— 小步快跑，每完成一个逻辑单元就提交。

5. 获取更新：git pull origin main —— 开发过程中，主分支可能有新代码，先合并到当前分支。

6. 推送到远程：git push origin feature-login —— 把分支推送到远程，方便代码审查。

7. 合并到主分支：在 GitHub 上发起 Pull Request，审查通过后合并到 main。

8. 清理分支：git branch -d feature-login —— 删除已合并的功能分支，保持仓库整洁。
