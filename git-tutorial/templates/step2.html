02 初始化仓库

git init、工作区与暂存区

场景什么时候初始化仓库

每当你开始一个新项目，或者想让一个已有项目被 Git 管理时，第一步就是初始化仓库。这会在项目根目录创建一个隐藏的 .git 文件夹，Git 的所有记录都保存在这里。

原理初始化就是建造档案室

执行 git init 时，Git 在你的项目文件夹里建造了一个精密的档案室——也就是 .git 目录。这个档案室有多个分区：对象库存放所有文件快照，引用区记录分支和标签，配置区保存仓库设置，钩子区存放自动化脚本。

初始化完成后，档案室是空的。它还没有收到任何文件，只是搭好了架子，等着你用 git add 和 git commit 往里面存东西。

.git 目录结构

要点初始化后 Git 在做什么

执行 git init 后，Git 会在当前目录创建 .git 子目录。这个目录包含了所有版本历史、分支信息、配置等。你不应该手动修改里面的内容。

初始化后，所有文件都处于「未跟踪」状态。Git 知道你目录里有哪些文件，但还没有开始记录它们的变化。你需要用 git add 告诉 Git 哪些文件要纳入版本控制。

错误常见错误

在错误的目录执行 git init，比如在「下载」或「桌面」这种包含很多无关文件的目录。结果 Git 把所有文件都标记为未跟踪，git status 输出一大串无关内容。

忘记配置用户名和邮箱就提交。Git 要求每次提交都记录作者信息，如果没有配置，提交会失败或显示匿名作者。
