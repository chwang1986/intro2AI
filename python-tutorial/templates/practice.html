综合实战

用 AI 协助从零写一个「待办事项管理器」命令行程序

场景从 0 到 1，和 AI 一起盖一座「小房子」

想象你要盖一座小房子。你不会自己烧砖、炼钢、做玻璃——而是准备好设计图，让专业的工人各司其职：地基队、框架队、装修队。你负责验收每个阶段，确保房子结实、实用、好看。

在这个实战项目中，我们要用 AI 协助开发一个完整的命令行待办事项管理器（Todo CLI）。它能添加任务、列出任务、标记完成、删除任务，数据持久化存储在 JSON 文件中。

这不是「让 AI 帮你写全部代码」的练习，而是「你指挥 AI，AI 出力，你把关」的实战演练。按照下面的五个步骤，体验完整的 AI 协作开发流程。

起源从命令行到分层架构

命令行界面（CLI）的历史几乎和计算机本身一样长。1960 年代，程序员通过电传打字机（Teletype）与计算机交互；1970 年代，Unix 操作系统普及了 Shell 和管道（pipe）的概念，让小型工具可以像搭积木一样组合。"Do one thing and do it well"（做好一件事）的 Unix 哲学，至今仍影响着 CLI 工具的设计。

分层架构的思想则来自企业级软件开发。1980 年代，随着数据库应用兴起，开发者们发现把"数据访问"、"业务规则"和"用户界面"混在一起会导致灾难。于是出现了三层架构：表现层（Presentation）、业务逻辑层（Business Logic）、数据访问层（Data Access）。这种分层让每一层只和相邻层交互，修改一层不会影响其他层。

我们的 Todo CLI 虽然很小，但也借鉴了这种分层思想：Task 是数据模型（表现数据层），TaskStorage 负责持久化（数据访问层），TaskManager 处理业务规则（业务逻辑层），main() 和 argparse 是用户界面（表现层）。这种设计让它容易扩展——比如你可以把 JSON 存储换成 SQLite，而业务逻辑层完全不用改。

思考为什么一个小项目也需要"架构"？

很多同学可能会想："Todo 程序不就几十行代码吗？有必要搞什么分层架构吗？"这种疑问非常合理。的确，对于"Hello World"级别的程序，分层是过度设计。但 Todo CLI 的目标是让你练习架构思维——就像数学课上的应用题，不一定每个场景都完全真实，但训练的思维是通用的。

假设你一开始把所有代码写在一个文件里，不分层：

# 混乱的单文件风格（反面教材）
def main():
    # 直接解析参数
    import sys
    cmd = sys.argv[1]
    
    # 直接操作文件
    import json
    if cmd == "add":
        with open("tasks.json") as f:
            tasks = json.load(f)
        tasks.append({"id": len(tasks)+1, "title": sys.argv[2]})
        with open("tasks.json", "w") as f:
            json.dump(tasks, f)
        print("添加了")
    # ... 更多混乱的代码

这种写法的问题在于：

1. 无法测试：业务逻辑和文件操作、用户输入完全混在一起，你很难单独测试"添加任务"这个逻辑。

2. 无法复用：如果你想给这个程序加一个 Web 界面，发现所有逻辑都和命令行参数绑死了，只能重写。

3. 无法协作：如果两个人同时开发，一个人改界面，一个人改存储格式，他们会不断修改同一文件，冲突频发。

分层架构就是用"现在的一点麻烦"换取"将来的很多不麻烦"。

论证为什么选 JSON、argparse 和分层？

在 Todo CLI 的设计中，我们做了几个关键选择。这些选择不是随意的，而是经过权衡的结果。让我们逐一论证。

1. JSON vs CSV vs SQLite

CSV 适合表格数据，但不适合嵌套结构。一个任务可能有多个标签、子任务，CSV 表达起来很别扭。SQLite 是正经的数据库，支持查询、索引、事务，但对于一个命令行小工具来说，引入 SQL 依赖和表结构迁移太重了。JSON 是" sweet spot "（最佳平衡点）：人类可读、Python 原生支持、能表达嵌套结构、不需要额外安装。代价是大数据量时查询慢，但对于 Todo 应用完全够用。

2. argparse 而不是 input() 或第三方库

用 input() 做交互式菜单最简单，但不适合脚本化调用（比如你想在自动化脚本里定时添加任务）。Click 和 Typer 等第三方库功能更强大，但需要额外安装。Python 标准库的 argparse 无需安装、文档完善、功能足够，是"零依赖"方案的最佳选择。

3. 为什么用 dataclass？

不用 dataclass，你也可以用字典表示任务：{"id": 1, "title": "买牛奶"}。但字典的问题是：没有类型检查、没有代码补全、容易打错键名。dataclass 让数据结构像普通类一样有属性访问，同时自动生成了 __init__ 等方法，既简洁又安全。

4. 反面论证：过度设计的边界

如果 Todo 程序要支持多用户协作、实时同步、Web 访问，那 JSON 文件肯定不够用，你需要数据库、网络层、权限系统。但不要为不存在的需求做设计——这就是软件工程中的 YAGNI 原则（You Aren't Gonna Need It，你不会需要它）。我们的分层架构已经预留了扩展空间，但不需要现在就实现。

代码完整实现参考

下面是把 AI 生成代码经过人工审查、整合、优化后的完整实现。把它保存为 todo.py，然后运行 python todo.py --help 开始体验。

代码示例
#!/usr/bin/env python3
"""
Todo CLI - 命令行待办事项管理器
由 AI 协助生成，经过人工审查和优化

使用方法：
    python todo.py add "买牛奶" --priority medium
    python todo.py list
    python todo.py list --filter completed
    python todo.py complete 1
    python todo.py delete 1
"""

import argparse
import json
import os
import sys
from dataclasses import asdict, dataclass, field
from datetime import datetime
from enum import Enum
from typing import Optional


# ============ 数据模型 ============

class Priority(Enum):
    LOW = "low"
    MEDIUM = "medium"
    HIGH = "high"

    def __str__(self):
        return self.value


@dataclass
class Task:
    id: int
    title: str
    priority: str = "medium"
    completed: bool = False
    created_at: str = field(default_factory=lambda: datetime.now().isoformat())

    def to_dict(self) -> dict:
        return asdict(self)

    @classmethod
    def from_dict(cls, data: dict) -> "Task":
        return cls(**data)


# ============ 存储层 ============

class TaskStorage:
    """JSON 文件持久化存储"""
    
    def __init__(self, filepath: str = "tasks.json"):
        self.filepath = filepath
    
    def load(self) -> list[Task]:
        """从 JSON 文件加载任务列表"""
        if not os.path.exists(self.filepath):
            return []
        
        try:
            with open(self.filepath, "r", encoding="utf-8") as f:
                data = json.load(f)
                if not isinstance(data, list):
                    print(f"⚠️  警告：{self.filepath} 格式异常，已重置为空列表")
                    return []
                return [Task.from_dict(item) for item in data]
        except json.JSONDecodeError:
            print(f"⚠️  警告：{self.filepath} 已损坏，已重置为空列表")
            return []
        except Exception as e:
            print(f"❌ 读取文件失败：{e}")
            return []
    
    def save(self, tasks: list[Task]) -> bool:
        """保存任务列表到 JSON 文件"""
        try:
            with open(self.filepath, "w", encoding="utf-8") as f:
                json.dump([t.to_dict() for t in tasks], f, ensure_ascii=False, indent=2)
            return True
        except Exception as e:
            print(f"❌ 保存文件失败：{e}")
            return False


# ============ 业务逻辑层 ============

class TaskManager:
    """任务管理器：负责所有业务逻辑"""
    
    def __init__(self, storage: TaskStorage):
        self.storage = storage
        self.tasks = self.storage.load()
        self._next_id = max((t.id for t in self.tasks), default=0) + 1
    
    def add(self, title: str, priority: str = "medium") -> Task:
        """添加新任务"""
        if not title or not title.strip():
            raise ValueError("❌ 任务标题不能为空")
        
        task = Task(
            id=self._next_id,
            title=title.strip(),
            priority=priority.lower()
        )
        self.tasks.append(task)
        self._next_id += 1
        self.storage.save(self.tasks)
        return task
    
    def list_tasks(self, filter_status: Optional[str] = None) -> list[Task]:
        """列出任务，支持按状态筛选"""
        result = self.tasks
        if filter_status == "pending":
            result = [t for t in result if not t.completed]
        elif filter_status == "completed":
            result = [t for t in result if t.completed]
        
        # 按优先级排序：high > medium > low
        priority_order = {"high": 0, "medium": 1, "low": 2}
        return sorted(result, key=lambda t: priority_order.get(t.priority, 99))
    
    def complete(self, task_id: int) -> bool:
        """标记任务为已完成"""
        for task in self.tasks:
            if task.id == task_id:
                if task.completed:
                    return False  # 已经是完成状态
                task.completed = True
                self.storage.save(self.tasks)
                return True
        return False
    
    def delete(self, task_id: int) -> bool:
        """删除任务"""
        for i, task in enumerate(self.tasks):
            if task.id == task_id:
                self.tasks.pop(i)
                self.storage.save(self.tasks)
                return True
        return False


# ============ 命令行界面 ============

# ANSI 颜色代码
COLOR_GREEN = "\033[32m"
COLOR_RED = "\033[31m"
COLOR_YELLOW = "\033[33m"
COLOR_RESET = "\033[0m"
BOLD = "\033[1m"


def color_priority(priority: str) -> str:
    """给优先级添加颜色"""
    colors = {
        "high": COLOR_RED,
        "medium": COLOR_YELLOW,
        "low": COLOR_GREEN,
    }
    color = colors.get(priority, "")
    return f"{color}{priority.upper()}{COLOR_RESET}"


def color_status(completed: bool) -> str:
    """给状态添加颜色"""
    if completed:
        return f"{COLOR_GREEN}✓ 已完成{COLOR_RESET}"
    return f"{COLOR_RED}○ 待办{COLOR_RESET}"


def print_tasks(tasks: list[Task]):
    """打印任务表格"""
    if not tasks:
        print("📭 暂无任务")
        return
    
    print(f"\n{BOLD}{'ID':<5} {'标题':<30} {'优先级':<10} {'状态':<12}{COLOR_RESET}")
    print("-" * 60)
    for task in tasks:
        title = task.title[:28] + ".." if len(task.title) > 30 else task.title
        print(f"{task.id:<5} {title:<30} {color_priority(task.priority):<18} {color_status(task.completed)}")
    print()


def main():
    parser = argparse.ArgumentParser(
        description="📝 命令行待办事项管理器",
        formatter_class=argparse.RawDescriptionHelpFormatter,
        epilog="""
示例：
  python todo.py add "学习 Python" --priority high
  python todo.py list
  python todo.py list --filter pending
  python todo.py complete 1
  python todo.py delete 1
        """
    )
    subparsers = parser.add_subparsers(dest="command", help="可用命令")
    
    # add 命令
    add_parser = subparsers.add_parser("add", help="添加新任务")
    add_parser.add_argument("title", help="任务标题")
    add_parser.add_argument("--priority", choices=["low", "medium", "high"],
                           default="medium", help="优先级（默认 medium）")
    
    # list 命令
    list_parser = subparsers.add_parser("list", help="列出所有任务")
    list_parser.add_argument("--filter", choices=["all", "pending", "completed"],
                            default="all", help="筛选条件（默认 all）")
    
    # complete 命令
    complete_parser = subparsers.add_parser("complete", help="标记任务为已完成")
    complete_parser.add_argument("task_id", type=int, help="任务 ID")
    
    # delete 命令
    delete_parser = subparsers.add_parser("delete", help="删除任务")
    delete_parser.add_argument("task_id", type=int, help="任务 ID")
    
    args = parser.parse_args()
    
    if not args.command:
        parser.print_help()
        sys.exit(0)
    
    # 初始化
    storage = TaskStorage()
    manager = TaskManager(storage)
    
    try:
        if args.command == "add":
            task = manager.add(args.title, args.priority)
            print(f"✅ 已添加任务 #{task.id}：{task.title} [{task.priority}]")
        
        elif args.command == "list":
            filter_map = {"all": None, "pending": "pending", "completed": "completed"}
            tasks = manager.list_tasks(filter_map[args.filter])
            status_label = {"all": "全部", "pending": "待办", "completed": "已完成"}
            print(f"\n📋 {status_label[args.filter]}任务列表：")
            print_tasks(tasks)
            print(f"共 {len(tasks)} 个任务")
        
        elif args.command == "complete":
            if manager.complete(args.task_id):
                print(f"✅ 任务 #{args.task_id} 已标记为完成")
            else:
                print(f"❌ 任务 #{args.task_id} 不存在或已是完成状态")
        
        elif args.command == "delete":
            if manager.delete(args.task_id):
                print(f"🗑️  任务 #{args.task_id} 已删除")
            else:
                print(f"❌ 任务 #{args.task_id} 不存在")
    
    except ValueError as e:
        print(f"❌ 输入错误：{e}")
        sys.exit(1)
    except Exception as e:
        print(f"❌ 发生错误：{e}")
        sys.exit(1)


if __name__ == "__main__":
    main()

错误实战中最常见的三个翻车现场

即使是这个小小的 Todo CLI，新手也容易在 AI 协作中踩坑：

1. 不做需求分析直接让 AI 写代码 —— 你脑子里只有一个模糊的想法"做个 Todo 程序"，就让 AI 开始写。结果 AI 生成的功能和你想的完全不一样——它没有命令行参数、没有持久化、不支持中文。先花 5 分钟把自己的需求写清楚，列出功能清单和约束条件，再让 AI 动手。

2. 不测试边界情况 —— 你运行了 python todo.py add "测试"，成功了，就以为万事大吉。但有没有试过添加 1000 个任务？有没有试过任务标题是 500 个字符？有没有在 JSON 文件中间删掉一个逗号再运行程序？边界测试是发现 AI 代码隐藏 Bug 的最有效手段。

3. 不处理异常 —— AI 生成的代码往往在「Happy Path」（正常路径）上运行良好，但一旦遇到异常就崩溃。磁盘满了无法写文件怎么办？用户输入了负数 ID 怎么办？JSON 文件被其他程序篡改了怎么办？每当你看到 AI 代码里没有 try-except 的地方，都要警惕。

# 当前 TaskStorage 使用 JSON 文件存储。
# 试着创建一个新的 MemoryStorage 类：

class MemoryStorage:
    def __init__(self):
        self._data = []
    
    def load(self):
        return self._data
    
    def save(self, tasks):
        self._data = tasks

# 然后修改 main() 函数：
# storage = TaskStorage()   # 原来
storage = MemoryStorage()   # 改成内存存储

# 运行程序，观察：
# 1. 程序还能正常工作吗？
# 2. 退出程序后，任务数据还在吗？
# 3. 这个实验证明了分层架构的什么优势？

实验结论：由于业务逻辑层（TaskManager）只依赖 Storage 的接口（load/save），而不关心具体实现，所以你可以轻松替换存储方式而不影响其他代码。这就是依赖注入和接口隔离的力量。

练习 4 思路提示
import sqlite3
from dataclasses import asdict

class SQLiteStorage:
    def __init__(self, db_path="tasks.db"):
        self.db_path = db_path
        self._init_table()
    
    def _init_table(self):
        with sqlite3.connect(self.db_path) as conn:
            conn.execute("""
                CREATE TABLE IF NOT EXISTS tasks (
                    id INTEGER PRIMARY KEY,
                    title TEXT,
                    priority TEXT,
                    completed INTEGER,
                    created_at TEXT
                )
            """)
    
    def load(self):
        with sqlite3.connect(self.db_path) as conn:
            rows = conn.execute("SELECT * FROM tasks").fetchall()
            return [Task(*row) for row in rows]  # 注意调整字段顺序
    
    def save(self, tasks):
        with sqlite3.connect(self.db_path) as conn:
            conn.execute("DELETE FROM tasks")
            for t in tasks:
                conn.execute(
                    "INSERT INTO tasks VALUES (?, ?, ?, ?, ?)",
                    (t.id, t.title, t.priority, int(t.completed), t.created_at)
                )
