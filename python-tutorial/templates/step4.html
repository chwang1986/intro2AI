函数

把重复代码打包成可复用的工具箱

场景为什么需要函数？

想象你是一名厨师，每天要做 100 杯珍珠奶茶。如果没有标准化流程，你每次都要想"先放多少茶、多少奶、煮多久珍珠"——既累又容易出错。于是你写了一张配方卡：只要给出杯数和甜度，就能自动产出一致的奶茶。

函数就是程序里的配方卡。同一段代码在多处使用时，与其复制粘贴十几遍，不如把它装进一个"有名字的盒子"，需要时喊一声名字就能用。

起源从数学函数到代码子程序

函数（Function）的概念最早来自数学。17 世纪，数学家莱布尼茨和牛顿在研究运动时，用 f(x) 表示「输入一个数 x，经过某种规则运算，输出一个结果」。比如 f(x) = x² 就是一个函数：输入 3，输出 9；输入 5，输出 25。这种「输入 → 处理 → 输出」的抽象方式，让数学家可以把复杂问题拆解成一个个可复用的模块。

1940 年代，第一台电子计算机诞生后，程序员很快发现：同样的计算逻辑在程序中反复出现。1950 年代初，一些汇编语言开始支持「子程序跳转」——你可以把一段常用代码放在某个位置，需要时跳过去执行，执行完再跳回来。1957 年的 Fortran 首次引入了正式的 SUBROUTINE（子程序）和 FUNCTION 概念，代码复用进入了高级语言时代。

到了 1960 年代，计算机科学家进一步提出了「模块化编程」和「过程式编程」的思想：程序应该由许多小而精的函数组成，每个函数只做一件事，并通过明确的接口（参数和返回值）相互协作。Python 的函数设计深受这一传统影响，同时又加入了现代特性——比如默认参数、关键字参数、多返回值解包等。

思考复制粘贴有什么问题

初学者最常犯的错误之一，就是遇到重复代码时选择「复制粘贴」。比如计算三个圆的面积，新手可能会写：

area1 = 3.14 * r1 * r1
area2 = 3.14 * r2 * r2
area3 = 3.14 * r3 * r3

这看起来简单直接，但如果后来发现 3.14 不够精确，需要用 3.14159，你就得把三行代码全部改一遍——漏改一行，程序就出了隐蔽的 bug。如果不止三处，而是三十处呢？

更严重的是，复制粘贴让代码的「意图」变得模糊。读到这三行时，你需要在脑子里解析一遍：「哦，这是在算圆面积」。而如果封装成 circle_area(r)，只要看一眼函数名，就知道它在做什么。函数不仅复用了代码，更复用了「理解」。

函数的另一个价值是抽象屏障。调用者不需要知道函数内部是怎么实现的，只需要知道「给它什么参数，它会返回什么结果」。就像你用手机拍照时，不需要理解 CMOS 传感器的工作原理——只要按下快门，就能得到照片。这种「黑盒」思维是管理复杂系统的核心能力。

原理函数的工作机制

函数是一段有名字的代码块，遵循"输入 → 处理 → 输出"的流水线模式。你可以把它看作一台自动贩卖机：从投币口塞入原料（参数），机器内部加工（执行代码），最后从取货口拿出成品（返回值）。

当程序调用函数时，Python 会在内存中开辟一块临时空间（称为栈帧），存放局部变量；函数执行完毕后，这块空间自动释放，返回值传回调用处。

论证为什么函数是组织代码的最佳粒度

既然函数这么好，那是不是把代码分得越细越好？或者反过来，把整个程序写成一个巨大的函数？这两种极端都有问题。函数的大小和数量，本质上是在「简洁性」和「可读性」之间找平衡。

太大的函数有什么问题？如果一个函数超过 50 行，它很可能在做不止一件事。阅读这样的代码时，你需要在脑子里记住很多临时变量和中间状态，认知负担很重。更麻烦的是，当你想复用其中的某一小部分逻辑时，发现它被紧耦合在大函数内部，抽不出来。

太碎的函数有什么问题？如果每两三行就封装成一个函数，代码会被淹没在大量的函数跳转中。读者需要不断切换上下文，去理解每个函数在做什么——最后反而比直接读 inline 代码更累。

业界有一个经验法则：一个函数应该只做「一件事」。判断标准是：你能不能用一个简单的动词短语给函数命名？比如 calculate_bmi()、validate_email()、sort_by_date()。如果命名变成了 process_data_and_save_and_notify()，说明这个函数做的事情太多了，应该拆分。

对比另一种代码组织方式——面向对象（类）——函数的优势在于简单和直接。你不需要理解「继承」「多态」「封装」这些概念，只需要知道「输入什么、输出什么」。对于初学者和中小规模项目来说，函数是最好上手的组织方式。Python 本身也是一门多范式语言，你可以先用函数写程序，等需要更复杂的结构时，再引入类。

要点函数的核心语法

def 关键字用来定义函数；参数是函数的"入口"，可以设定默认值；return 是函数的"出口"，没有 return 默认返回 None。变量在函数内部创建的是局部变量，只在函数内可见。

代码示例
# 1. 基本函数定义与调用
def greet(name):
    """向用户打招呼"""
    message = f"你好，{name}！"
    return message

print(greet("小明"))   # 输出: 你好，小明！


# 2. 默认参数：给参数设置默认值
def make_coffee(type="美式", sugar=0):
    return f"一杯{type}，{sugar}勺糖"

print(make_coffee())               # 一杯美式，0勺糖
print(make_coffee("拿铁", 2))      # 一杯拿铁，2勺糖
print(make_coffee(sugar=1))        # 一杯美式，1勺糖（关键字传参）


# 3. return 返回多个值（实际是元组）
def calc(a, b):
    return a + b, a - b, a * b

sum_val, diff, prod = calc(10, 3)
print(sum_val, diff, prod)   # 13 7 30


# 4. 作用域：局部变量 vs 全局变量
count = 0   # 全局变量

def increment():
    global count   # 声明使用全局变量
    count += 1
    local_var = "我只能在函数内使用"
    return count

increment()
print(count)   # 1

实验亲手探索函数的边界行为

实验 1：验证局部变量的生命周期

运行下面的代码，观察报错信息，理解局部变量和全局变量的区别：

count = 100   # 全局变量

def demo():
    count = 5   # 这是局部变量，和外面的 count 不是同一个！
    print("函数内部:", count)

demo()
print("函数外部:", count)   # 仍然是 100

# 如果你想在函数里修改全局变量，需要用 global
def demo2():
    global count
    count = 99
    print("函数内部:", count)

demo2()
print("函数外部:", count)   # 变成了 99

实验 2：观察默认参数的「陷阱」

这个实验会让你大吃一惊——运行下面两段代码，对比输出：

# 实验 A：默认参数是数字（不可变）
def add_one(n=0):
    n += 1
    return n

print(add_one())   # 1
print(add_one())   # 1  —— 每次都是 1，符合预期

# 实验 B：默认参数是列表（可变）
def add_item(item, items=[]):
    items.append(item)
    return items

print(add_item(1))   # [1]
print(add_item(2))   # [1, 2]  —— 糟糕！列表记住了上次的内容！

为什么会这样？因为默认参数在函数定义时就被创建了一次，之后每次调用都复用同一个对象。数字、字符串等不可变对象被修改时会创建新对象，所以看起来「正常」；但列表是可变的，append() 直接修改了原来的列表。正确的做法是用 None 作为默认值。

实验 3：体验递归函数

函数可以调用自己，这叫做递归。试着运行下面的阶乘函数，并在关键位置加入 print，观察调用栈的形成和消退：

def factorial(n):
    print(f"计算 factorial({n})")
    if n <= 1:
        print(f"factorial({n}) 的基础情况，返回 1")
        return 1
    result = n * factorial(n - 1)
    print(f"factorial({n}) = {n} * {result // n} = {result}")
    return result

factorial(4)

观察输出，你会发现计算是从 factorial(4) 开始，逐层深入到 factorial(1)，然后再逐层返回结果。这个过程就像剥洋葱：先一层层剥进去，到达中心后再一层层剥出来。

错误常见陷阱

1. 忘记 return：函数没有 return 时默认返回 None，新手常误以为会得到计算结果。

def add(a, b):
    result = a + b
    # 糟糕！忘记写 return result

print(add(2, 3))   # 输出 None，不是 5

2. 在函数外访问局部变量：局部变量随函数结束而销毁，外部无法访问。

def test():
    x = 10

test()
print(x)   # NameError: name 'x' is not defined

3. 默认参数用可变对象：默认参数在函数定义时只创建一次，使用列表或字典作为默认值会导致意外累积。

# 错误示范
def append_item(item, items=[]):
    items.append(item)
    return items

print(append_item(1))   # [1]
print(append_item(2))   # [1, 2]  —— 糟糕，列表记住了上次的结果！

# 正确做法
def append_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items

def calculate_bmi(weight, height):
    bmi = weight / (height ** 2)
    if bmi < 18.5:
        category = "偏瘦"
    elif bmi < 24:
        category = "正常"
    elif bmi < 28:
        category = "超重"
    else:
        category = "肥胖"
    return bmi, category

bmi, cat = calculate_bmi(70, 1.75)
print(f"BMI: {bmi:.1f}, 分类: {cat}")
