面向对象

用类和对象组织复杂的代码世界

场景代码膨胀了怎么办？

假设你要开发一个宠物医院管理系统。如果只用函数，你会写出一堆零散的 feed_dog()、cat_vaccine()、rabbit_age()……数据和操作分离，越来越乱。

面向对象的思路是：把数据（属性）和操作数据的方法（行为）打包到一个"模板"里。就像玩具工厂先有设计图纸（类），再按图纸生产出一个个玩具（对象）。狗有名字、年龄、会叫；猫也有名字、年龄、但会喵喵叫——它们共享"宠物"的模板，又有各自的特色。

原理类、对象与继承

类（Class）是对象的蓝图，定义了这类事物有什么属性、能做什么。对象（Object）是类的实例，是内存中真实存在的数据。继承（Inheritance）则让子类复用父类的代码，同时扩展自己的特性，避免重复造轮子。

self 是对象自身的引用，让每个对象能访问自己的属性和方法。它像每个对象随身携带的"身份证"，方法通过它找到"属于我自己的数据"。

图解

要点面向对象的核心语法

class 定义类；__init__ 是构造方法，在创建对象时自动执行；self 代表实例自身，必须通过它访问实例属性；继承在类名后的括号中指定父类，子类自动拥有父类的全部属性和方法。

代码示例
# 1. 定义类和构造方法
class Animal:
    """动物基类"""
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def speak(self):
        return f"{self.name} 发出声音"

    def introduce(self):
        return f"我叫{self.name}，今年{self.age}岁"


# 2. 继承：Dog 继承 Animal
class Dog(Animal):
    def __init__(self, name, age, breed):
        super().__init__(name, age)   # 调用父类构造方法
        self.breed = breed            # 子类自己的属性

    def speak(self):
        return f"{self.name} 说：汪汪！"   # 重写父类方法

    def fetch(self):
        return f"{self.name} 去捡球了"


# 3. Cat 也继承 Animal
class Cat(Animal):
    def __init__(self, name, age, color):
        super().__init__(name, age)
        self.color = color

    def speak(self):
        return f"{self.name} 说：喵喵~"

    def climb(self):
        return f"{self.name} 爬上树了"


# 4. 创建对象（实例化）
dog = Dog("旺财", 3, "金毛")
cat = Cat("咪咪", 2, "橘色")

print(dog.introduce())   # 我叫旺财，今年3岁（继承自 Animal）
print(dog.speak())       # 旺财 说：汪汪！（子类重写）
print(dog.fetch())       # 旺财 去捡球了（子类新增）

print(cat.speak())       # 咪咪 说：喵喵~


# 5. 多态：同一消息，不同对象不同响应
animals = [Dog("A", 1, "柯基"), Cat("B", 2, "白")]
for animal in animals:
    print(animal.speak())   # 各自发出不同的声音

错误常见陷阱

1. 忘记写 self：实例方法的第一个参数必须是 self，代表对象自身。忘记写会导致 TypeError。

class Person:
    def __init__(name):   # 错误！漏了 self
        self.name = name

# 正确
class Person:
    def __init__(self, name):
        self.name = name

2. 方法名和属性名冲突：不要用相同的名字定义方法和属性，否则后定义的会覆盖前者。

class Circle:
    def __init__(self, radius):
        self.radius = radius

    def area(self):           # 方法
        return 3.14 * self.radius ** 2

# 不要这样做：
c = Circle(5)
c.area = 100   # 糟糕！把方法 area 覆盖成了整数 100
# 现在 c.area() 会报错：'int' object is not callable

3. 过度使用继承："是一个"关系才用继承（Dog 是 Animal），"有一个"关系应该用组合（Car 有一个 Engine）。滥用继承会让类关系像意大利面条一样纠缠不清。

# 错误的继承：企鹅不是典型的鸟（不会飞）
class Bird:
    def fly(self): ...

class Penguin(Bird):   # 企鹅继承鸟？
    def fly(self):       # 被迫重写为空或抛异常
        raise Exception("我不会飞")

# 正确的组合：把"能否飞"拆成行为对象
class FlyBehavior:
    def fly(self): ...

class CannotFly(FlyBehavior):
    def fly(self): return "不会飞"

class Bird:
    def __init__(self, fly_behavior):
        self.fly_behavior = fly_behavior
