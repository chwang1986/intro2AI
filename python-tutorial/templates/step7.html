面向对象

用类和对象组织复杂的代码世界

场景代码膨胀了怎么办？

假设你要开发一个宠物医院管理系统。如果只用函数，你会写出一堆零散的 feed_dog()、cat_vaccine()、rabbit_age()……数据和操作分离，越来越乱。

面向对象的思路是：把数据（属性）和操作数据的方法（行为）打包到一个"模板"里。就像玩具工厂先有设计图纸（类），再按图纸生产出一个个玩具（对象）。狗有名字、年龄、会叫；猫也有名字、年龄、但会喵喵叫——它们共享"宠物"的模板，又有各自的特色。

起源从仿真语言到编程范式

面向对象编程（OOP）的雏形诞生于 1962 年的挪威。计算机科学家 Ole-Johan Dahl 和 Kristen Nygaard 在开发仿真语言 Simula 时，为了让程序更好地模拟现实世界（如港口船只调度），首次提出了"类"（Class）和"对象"（Object）的概念。他们把船的属性（吨位、速度）和行为（进港、出港）打包在一起，这就是类的原型。

1970 年代，Xerox PARC 研究中心的 Alan Kay 在设计 Smalltalk 语言时，将面向对象理念发扬光大，并创造了"面向对象编程"这个术语。Smalltalk 还催生了图形用户界面（GUI）的雏形——你看到的窗口、图标、鼠标，都和 OOP 思想同源。

1980 年代，Bjarne Stroustrup 在 C 语言基础上增加了类机制，创造了 C++，让系统级编程也能享受 OOP 的好处。1995 年，Sun 公司发布 Java，喊出"一次编写，到处运行"的口号，把 OOP 推向了企业级开发的主流。而 Python（1991 年）则走了一条更灵活的路——它支持 OOP，但不强制你使用，你可以根据场景自由选择过程式、函数式或面向对象风格。

思考当代码膨胀到失控，我们该怎么办？

1960 年代的软件危机让很多程序员夜不能寐。当时一个典型场景是：你写了一个程序管理学生信息，数据存在全局变量里，一堆函数操作这些变量。起初只有几百行，还能看懂。但随着功能增加——要加教师信息、课程信息、成绩管理——代码膨胀到几万行，变量名开始冲突，修改一个函数可能莫名其妙地破坏了另一个功能。

这个过程式编程的困境可以总结为三个痛点：

1. 数据和函数分离：学生的姓名、年龄散落在各处，任何函数都能修改它们。你根本不知道哪个函数在什么时候改了什么数据。

2. 全局变量灾难：大型程序里，变量名越来越长（student_name_temp_final），冲突越来越频繁。

3. 复用困难：你想把"学生管理"代码搬到另一个项目里，发现它和"课程管理"深度耦合，根本无法拆分，只能复制粘贴，然后维护两份代码。

面向对象的出现，正是为了把混乱的代码世界模块化、封装化。每个类像一个独立的小王国，管理自己的数据，提供对外的接口。你想用？创建一个实例就行。你想复用？继承或组合就行。

原理类、对象与继承

类（Class）是对象的蓝图，定义了这类事物有什么属性、能做什么。对象（Object）是类的实例，是内存中真实存在的数据。继承（Inheritance）则让子类复用父类的代码，同时扩展自己的特性，避免重复造轮子。

self 是对象自身的引用，让每个对象能访问自己的属性和方法。它像每个对象随身携带的"身份证"，方法通过它找到"属于我自己的数据"。

论证为什么 OOP 能解决代码组织问题？

有人可能会说："我用函数也能写出好代码啊，为什么非要学 OOP？"没错，对于小程序，过程式完全够用。但当项目超过一定规模，OOP 的组织优势就会显现。让我们从几个角度论证：

1. 封装 vs 裸数据

过程式编程中，数据是裸露的，任何函数都能修改。OOP 通过封装把数据保护起来，只允许通过规定的方法访问。就像银行的金库——你不直接进去数钱，而是通过柜台办理业务。这样可以在方法里加入校验逻辑（比如余额不能为负数），保证数据始终有效。

2. 继承 vs 复制粘贴

假设 Dog 和 Cat 都有 name、age 和 eat() 方法。过程式编程里，你要写两套几乎一样的代码。OOP 的继承让你把共性放到 Animal 父类里，子类只写差异部分。不仅减少了重复代码，而且当共性需要修改时（比如给 eat() 增加"进食日志"），只需改父类一处。

3. 多态 vs 条件分支

过程式编程处理"不同类型不同行为"时，通常要写一长串 if/else：

# 过程式风格（痛苦）
def make_sound(animal_type, name):
    if animal_type == "dog":
        print(f"{name} 汪汪")
    elif animal_type == "cat":
        print(f"{name} 喵喵")
    elif animal_type == "duck":
        print(f"{name} 嘎嘎")
    # 每增加一种动物，就要修改这个函数！

OOP 的多态让每种动物自己决定怎么叫，调用者无需关心具体类型。增加新动物时，旧代码完全不需要修改——这就是著名的开闭原则（对扩展开放，对修改关闭）。

4. 但 OOP 不是银弹

Python 不像 Java 那样强制所有代码都包在类里。如果你只是写个简单的脚本，几行函数就够了。过度使用 OOP（比如给每个小功能都创建一个类）反而会让代码臃肿。好的设计是选择合适的工具，而不是把工具当成信仰。

要点面向对象的核心语法

class 定义类；__init__ 是构造方法，在创建对象时自动执行；self 代表实例自身，必须通过它访问实例属性；继承在类名后的括号中指定父类，子类自动拥有父类的全部属性和方法。

代码示例
# 1. 定义类和构造方法
class Animal:
    """动物基类"""
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def speak(self):
        return f"{self.name} 发出声音"

    def introduce(self):
        return f"我叫{self.name}，今年{self.age}岁"


# 2. 继承：Dog 继承 Animal
class Dog(Animal):
    def __init__(self, name, age, breed):
        super().__init__(name, age)   # 调用父类构造方法
        self.breed = breed            # 子类自己的属性

    def speak(self):
        return f"{self.name} 说：汪汪！"   # 重写父类方法

    def fetch(self):
        return f"{self.name} 去捡球了"


# 3. Cat 也继承 Animal
class Cat(Animal):
    def __init__(self, name, age, color):
        super().__init__(name, age)
        self.color = color

    def speak(self):
        return f"{self.name} 说：喵喵~"

    def climb(self):
        return f"{self.name} 爬上树了"


# 4. 创建对象（实例化）
dog = Dog("旺财", 3, "金毛")
cat = Cat("咪咪", 2, "橘色")

print(dog.introduce())   # 我叫旺财，今年3岁（继承自 Animal）
print(dog.speak())       # 旺财 说：汪汪！（子类重写）
print(dog.fetch())       # 旺财 去捡球了（子类新增）

print(cat.speak())       # 咪咪 说：喵喵~


# 5. 多态：同一消息，不同对象不同响应
animals = [Dog("A", 1, "柯基"), Cat("B", 2, "白")]
for animal in animals:
    print(animal.speak())   # 各自发出不同的声音

实验亲手揭开 self 和继承的面纱

打开 Python 交互式环境，跟着下面的实验一步步操作，观察输出结果，理解背后的原理。

实验 1：self 到底是什么？

class Demo:
    def show_self(self):
        print(f"self 的值是：{self}")
        print(f"self 的 id 是：{id(self)}")

d1 = Demo()
d2 = Demo()

print(f"d1 的 id 是：{id(d1)}")
print(f"d2 的 id 是：{id(d2)}")

d1.show_self()  # 观察：self 就是 d1！
d2.show_self()  # 观察：self 就是 d2！

# 甚至可以直接这样调用：
Demo.show_self(d1)  # 等价于 d1.show_self()

实验结论：self 就是实例对象本身。当你调用 d1.show_self() 时，Python 自动把 d1 作为第一个参数传给方法。

实验 2：继承时方法怎么找？

class Animal:
    def speak(self):
        return "动物发出声音"

class Dog(Animal):
    def speak(self):
        return "汪汪"

class Cat(Animal):
    pass  # 不重写 speak

d = Dog()
c = Cat()

print(d.speak())  # 汪汪 —— 调用 Dog 自己的方法
print(c.speak())  # 动物发出声音 —— 去父类找！

# 查看方法解析顺序（MRO）
print(Dog.__mro__)  # 输出查找顺序

实验结论：当调用 c.speak() 时，Python 先在 Cat 类里找，找不到就去父类 Animal 找。这个查找顺序可以通过 __mro__ 查看。

实验 3：super() 是怎么工作的？

class Parent:
    def __init__(self):
        print("Parent.__init__ 被调用")
        self.parent_attr = "父类属性"

class Child(Parent):
    def __init__(self):
        print("Child.__init__ 被调用")
        super().__init__()  # 显式调用父类构造方法
        self.child_attr = "子类属性"

c = Child()
print(c.parent_attr)  # 能访问到吗？
print(c.child_attr)   # 能访问到吗？

实验结论：如果不写 super().__init__()，子类实例就不会拥有父类在 __init__ 中定义的属性。super() 像一个"父类代理"，帮你调用父类的方法。

错误常见陷阱

1. 忘记写 self：实例方法的第一个参数必须是 self，代表对象自身。忘记写会导致 TypeError。

class Person:
    def __init__(name):   # 错误！漏了 self
        self.name = name

# 正确
class Person:
    def __init__(self, name):
        self.name = name

2. 方法名和属性名冲突：不要用相同的名字定义方法和属性，否则后定义的会覆盖前者。

class Circle:
    def __init__(self, radius):
        self.radius = radius

    def area(self):           # 方法
        return 3.14 * self.radius ** 2

# 不要这样做：
c = Circle(5)
c.area = 100   # 糟糕！把方法 area 覆盖成了整数 100
# 现在 c.area() 会报错：'int' object is not callable

3. 过度使用继承："是一个"关系才用继承（Dog 是 Animal），"有一个"关系应该用组合（Car 有一个 Engine）。滥用继承会让类关系像意大利面条一样纠缠不清。

# 错误的继承：企鹅不是典型的鸟（不会飞）
class Bird:
    def fly(self): ...

class Penguin(Bird):   # 企鹅继承鸟？
    def fly(self):       # 被迫重写为空或抛异常
        raise Exception("我不会飞")

# 正确的组合：把"能否飞"拆成行为对象
class FlyBehavior:
    def fly(self): ...

class CannotFly(FlyBehavior):
    def fly(self): return "不会飞"

class Bird:
    def __init__(self, fly_behavior):
        self.fly_behavior = fly_behavior

class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount
        print(f"存入 {amount} 元，当前余额: {self.balance}")

    def withdraw(self, amount):
        if amount > self.balance:
            print("余额不足")
        else:
            self.balance -= amount
            print(f"取出 {amount} 元，当前余额: {self.balance}")

    def get_balance(self):
        return self.balance

account = BankAccount("小明", 1000)
account.deposit(500)
account.withdraw(300)
print(account.get_balance())
练习 4 思路提示
import math

class Shape:
    def area(self):
        raise NotImplementedError("子类必须实现 area 方法")
    
    def perimeter(self):
        raise NotImplementedError("子类必须实现 perimeter 方法")

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height
    
    def perimeter(self):
        return 2 * (self.width + self.height)

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        return math.pi * self.radius ** 2
    
    def perimeter(self):
        return 2 * math.pi * self.radius

def print_shape_info(shape):
    print(f"面积: {shape.area():.2f}, 周长: {shape.perimeter():.2f}")

# 多态：同一个函数，处理不同类型的对象
print_shape_info(Rectangle(3, 4))
print_shape_info(Circle(5))

测验快速检验一下

1. 单选题：Python 中类的方法第一个参数为什么通常叫 self？

A. 这是 Python 关键字，必须用 self

B. 这只是约定，可以用任意名字，但 self 是社区惯例

C. self 是 Python 内部保留的特殊对象

D. 不写 self 程序也能正常运行

点击查看答案

正确答案：B。self 不是关键字，只是一个命名约定。你可以用 this、me 或其他名字，但会让其他程序员困惑。

2. 判断题：子类不定义 __init__ 方法时，会自动调用父类的 __init__。

A. 正确

B. 错误，子类如果不写 __init__，实例就没有任何属性

点击查看答案

正确答案：A。如果子类没有定义 __init__，Python 会沿着继承链找到父类的 __init__ 并调用它。

3. 单选题：以下哪种关系适合用继承（is-a）而不是组合（has-a）？

A. 汽车 和 引擎

B. 学生 和 课程

C. 猫 和 动物

D. 电脑 和 显示器

点击查看答案

正确答案：C。"猫是一种动物"是继承关系。其他选项都是"有一个"的关系，应该用组合。

4. 单选题：super().__init__() 的作用是？

A. 创建一个新的父类实例

B. 调用父类的 __init__ 方法，让子类也能拥有父类定义的属性

C. 返回父类的引用

D. 终止子类的初始化

点击查看答案

正确答案：B。super() 返回一个代理对象，让你可以调用父类的方法。在 __init__ 中调用它是为了复用父类的初始化逻辑。

5. 判断题：Python 强制所有代码必须使用面向对象风格。

A. 正确，Python 是纯面向对象语言

B. 错误，Python 支持多种编程范式，你可以只用函数写完整程序

点击查看答案

正确答案：B。Python 是多范式语言，支持过程式、函数式和面向对象编程。不像 Java，Python 不强制你使用类。
