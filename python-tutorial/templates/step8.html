调用 AI API

让你的程序学会「开口说话」——与人工智能对话

场景程序也需要「打电话」

想象你是一位忙碌的总经理，每天需要处理大量信息：总结会议记录、翻译外文邮件、分析客户反馈。你不可能事事亲为，于是你拨通了一位超级助手的电话——这位助手就是 AI 模型。

在编程世界里，调用 AI API 就像给这位助手打电话。你的程序拨通号码（发送 HTTP 请求），告诉助手要做什么（传递 Prompt），助手思考后给出回答（返回 JSON 响应）。你只需要学会「打电话」的正确姿势，就能让程序拥有理解语言、生成内容、分析情感的超能力。

例如，你写了一个自动总结文章的小工具：程序读取一篇长文，「打给」AI 说"请用三句话总结以下内容"，AI 立刻返回精炼的摘要——全程无需人工干预。

原理一次 API 调用的旅程

当你调用 AI API 时，数据经历了怎样的旅程？就像寄一封特快专递：

你的程序把请求写好（信）→ HTTP 协议负责运输（快递车）→ API 网关检查身份并分拣（邮局）→ AI 模型处理思考（收件人读信写回信）→ 原路返回 JSON 响应（回信送达）。

API Key 相当于你的「身份证」，证明你有权使用这项服务。JSON 则是通用的「信封格式」，无论里面装的是文本、数字还是列表，都能被双方理解。

要点如何正确「拨打电话」

调用 AI API 的核心技能包括：使用 requests 库发送 HTTP 请求、安全地管理 API Key、构造正确的请求体、解析 JSON 响应，以及优雅地处理各种异常情况。

下面的示例展示了如何安全地调用 OpenAI API 进行文本总结。注意我们把 API Key 存放在环境变量中，而不是直接写在代码里——就像你不会把家门钥匙贴在门牌上一样。

代码示例
import os
import requests
from dotenv import load_dotenv

# 1. 从环境变量加载 API Key（安全第一！）
load_dotenv()
API_KEY = os.getenv("OPENAI_API_KEY")
API_URL = "https://api.openai.com/v1/chat/completions"

def summarize_article(article_text):
    """调用 AI API 总结文章，像打电话一样礼貌而清晰"""
    
    # 2. 构造请求头：带上你的「身份证」
    headers = {
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    }
    
    # 3. 构造请求体：告诉 AI 要做什么
    payload = {
        "model": "gpt-4o-mini",
        "messages": [
            {"role": "system", "content": "你是一位擅长总结的编辑。"},
            {"role": "user", "content": f"请用三句话总结以下文章：\n\n{article_text}"}
        ],
        "temperature": 0.7,
        "max_tokens": 150
    }
    
    try:
        # 4. 发送 POST 请求（拨号）
        response = requests.post(API_URL, json=payload, headers=headers, timeout=30)
        
        # 5. 检查 HTTP 状态码
        response.raise_for_status()
        
        # 6. 解析 JSON 响应（读取回信）
        data = response.json()
        summary = data["choices"][0]["message"]["content"]
        return summary
        
    except requests.exceptions.Timeout:
        return "错误：请求超时，AI 助手可能太忙了，请稍后再试。"
    except requests.exceptions.ConnectionError:
        return "错误：网络连接失败，请检查你的网络。"
    except requests.exceptions.HTTPError as e:
        if response.status_code == 429:
            return "错误：请求太频繁了，请放慢速度（API 限流）。"
        return f"错误：HTTP {response.status_code} - {e}"
    except KeyError:
        return "错误：响应格式异常，无法提取总结内容。"
    except Exception as e:
        return f"错误：未知异常 - {e}"

# 使用示例
article = """
Python 是一种高级编程语言，由 Guido van Rossum 于 1991 年创建。
它以简洁、易读的语法著称，广泛应用于数据分析、人工智能、Web 开发等领域。
Python 拥有庞大的第三方库生态系统，让开发者能够快速构建复杂应用。
"""

result = summarize_article(article)
print("文章总结：")
print(result)

错误这些坑千万别踩

新手调用 AI API 时，总会犯一些「低级错误」。下面三个是最常见、也最危险的：

1. 硬编码 API Key —— 把密钥直接写在代码里，然后不小心提交到了 GitHub。这就像把银行卡密码写在名片上到处发。攻击者拿到你的 Key 后，可以疯狂调用 API，让你一夜之间欠下巨额账单。务必使用环境变量或专用密钥管理服务。

2. 不处理网络异常 —— 以为网络永远畅通无阻。现实中，WiFi 会断、服务器会挂、DNS 会失效。如果不加 try-except 和超时设置，程序遇到一点网络波动就会崩溃，用户体验极差。永远为网络请求加上异常处理和超时参数。

3. 忽略 API 限流（Rate Limit） —— 像机关枪一样连续发送请求，结果被服务商封禁。API 提供商通常限制每分钟/每小时的调用次数。不处理 429 状态码，不仅会导致请求失败，还可能触发账号风控。实现指数退避重试机制，遇到限流时主动等待。
