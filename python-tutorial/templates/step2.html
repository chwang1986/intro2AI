02 变量与类型

数据存储的基本方式，理解变量和类型

起源从「命名盒子」到变量概念

变量（Variable）的概念可以追溯到 19 世纪数学家对代数的研究。在方程 x + 3 = 5 中，x 是一个「占位符」，代表一个我们暂时不知道但可以通过计算求出的值。这种思想被计算机科学家们直接借用到了编程中。

1945 年，冯·诺依曼提出了现代计算机体系结构：程序和数据都存放在同一个存储器中。这意味着程序运行时需要在内存中「标记」某些位置，方便后续读取和修改。这些标记就是变量名。

早期的编程语言（如汇编）直接用内存地址操作数据，比如「把 100 号地址的值加 1」。1957 年诞生的 Fortran 是第一个引入「变量名」的高级语言，程序员终于可以用 age 而不是 0x1A2B 来指代数据了。这看似小小的改进，实际上让编程从「机器思维」迈入了「人类思维」。

思考为什么数据需要分类（类型）

假设你拿到一盒积木，里面有不同形状的木块。如果不加区分，你可能会把圆形块硬塞进方形孔里——结果当然是卡住。计算机处理数据时也面临同样的问题。

在计算机底层，所有数据都是一串 0 和 1。同样 32 个二进制位，可以被解读为整数（比如 65）、浮点数（比如 1.8e-44）、或者一个字符（字母 'A' 的 ASCII 码就是 65）。如果不告诉计算机「这串二进制应该按什么规则解读」，它就会做出错误的理解。

类型系统（Type System）就是解决这个问题的方案。它给每种数据贴上「标签」：这是整数、那是文字、这是真假值。不同的标签决定了数据能参与哪些运算——你可以对整数做加减，对字符串做拼接，但对字符串做除法就毫无意义。类型系统的本质是一套规则和保护机制，确保数据被正确使用。

场景程序需要记住数据

想象你在整理房间。你有一些东西要存放——书、衣服、工具。你不会把它们随便扔在地上，而是会放进贴好标签的盒子里：「书籍」「冬衣」「螺丝刀」。当你需要某样东西时，只要看标签就能找到对应的盒子。

程序处理数据也是一样的道理。用户输入的名字、计算得到的分数、从网络下载的图片——这些都需要先存起来才能使用。变量就是程序里的「贴标签的盒子」，而类型则规定了盒子里能放什么东西。

比如 name = "小明" 就是创建了一个标签为 name 的盒子，里面放了一段文字。以后程序看到 name 这个标签，就知道去哪里取这段文字。

原理变量是指向数据的标签

在 Python 中，变量不是「装数据的盒子」，而是「贴在数据上的标签」。更准确地说，变量是一个指向内存地址的指针。当你写 a = 10 时，Python 先在内存中创建一个整数对象 10，然后让变量 a 指向这个对象。

这与「静态类型语言」（如 C、Java）截然不同。在静态类型语言中，你必须先声明「a 是整数类型」，然后才能赋值。而在 Python 这样的动态类型语言中，类型由赋值的数据自动决定，同一个变量可以先指向整数，再指向字符串。

这种设计带来了极大的灵活性，但也意味着你需要清楚地知道自己手中的数据是什么类型，否则会在运行时遇到类型错误。

论证为什么 Python 选择动态类型

编程语言在类型系统上有两个主要阵营：静态类型（如 C、Java、Go）和动态类型（如 Python、JavaScript）。Python 选择了动态类型，这不是偶然，而是有其深刻的设计考量。

静态类型的逻辑是「防患于未然」：你在写代码时就要声明 int age = 25，编译器会在运行前检查所有类型是否匹配。优点是能在编译阶段发现很多错误，运行速度通常也更快。缺点是代码更冗长，灵活性差——如果你想让同一个变量先存整数再存字符串，就必须重新声明或转换类型。

动态类型的逻辑是「灵活至上」：变量本身没有类型，类型属于数据对象。你可以先写 x = 10，再写 x = "hello"，解释器在运行时自动判断。这让代码更简洁，特别适合快速原型开发、数据分析和脚本编写——这些场景里，数据的形态往往在运行时才确定。

对于初学者来说，动态类型降低了入门门槛。你不需要一开始就理解「类型声明」「编译期检查」这些概念，可以专注于「程序在做什么」。当然，灵活也是一把双刃剑：当项目变大时，类型错误可能在运行时才发现。这也是为什么 Python 3.5 引入了「类型提示」（Type Hints），让有经验的开发者可以在保持动态类型灵活性的同时，增加代码的可维护性。

实验亲手验证变量与类型的秘密

实验 1：用 id() 查看变量的内存地址

Python 中每个对象都有唯一的内存地址。用 id() 函数可以看到变量指向的地址，验证「变量是标签而不是盒子」：

a = 10
b = 10
print(id(a))   # 比如: 140735893619024
print(id(b))   # 和上面一模一样！
print(a is b)  # True —— 它们指向同一个对象

a = 20
print(id(a))   # 地址变了！说明 a 现在指向了另一个对象
print(b)       # b 还是 10，不受影响

对于小整数（-5 到 256），Python 会缓存复用，所以 a 和 b 指向同一个对象。这个实验证明了变量只是「指向对象的标签」。

实验 2：观察浮点数的精度陷阱

在交互式解释器中输入下面这行代码，看看结果是否出乎你意料：

>>> 0.1 + 0.2
0.30000000000000004

>>> 0.1 + 0.2 == 0.3
False

这不是 Python 的 bug，而是计算机用二进制存储浮点数的固有局限。就像十进制无法精确表示 1/3（0.3333...），二进制也无法精确表示 0.1。这就是为什么金融计算通常用整数（比如把金额存为「分」而不是「元」）或专门的 decimal 模块。

实验 3：类型转换的边界测试

试着运行下面这些转换，观察哪些成功、哪些报错，理解报错信息在告诉你什么：

int("100")       # 成功 → 100
int("100.5")     # 报错！不能直接把带小数点的字符串转整数
int(float("100.5"))  # 先转 float 再转 int → 100
float("abc")     # 报错！无法解析
str(123)         # 成功 → "123"
bool(0)          # 成功 → False
bool("")         # 成功 → False
bool("False")    # 成功 → True！非空字符串就是 True

要点基本类型与类型检查

Python 有四种最常用的基础类型：int（整数）、float（浮点数）、str（字符串）、bool（布尔值）。你可以用 type() 函数随时查看一个变量的类型。

类型转换也很常用：用 int() 把字符串转成整数，str() 把数字转成字符串，float() 把整数转成小数。但转换时要注意——int("abc") 会报错，因为 "abc" 根本不是一个数字。

代码示例
# 四种基础类型
age = 25              # int 整数
height = 1.75         # float 浮点数
name = "Alice"        # str 字符串
is_student = True     # bool 布尔值 (True / False)

# 查看类型
print(type(age))      # <class 'int'>
print(type(name))     # <class 'str'>

# 类型转换
s = "100"
num = int(s)          # 字符串转整数
print(num + 1)        # 101

pi_str = str(3.14)    # 数字转字符串
print("pi = " + pi_str)  # pi = 3.14

错误初学者常犯的错误

字符串和数字直接相加："年龄：" + 18 会报错，因为 Python 不知道该怎么把字符串和数字拼在一起。正确的做法是先把数字转成字符串："年龄：" + str(18)，或者使用 f-string：f"年龄：{18}"。

用 == 比较浮点数：由于计算机存储浮点数存在精度误差，0.1 + 0.2 == 0.3 的结果是 False！正确的做法是判断两个浮点数的差是否小于一个很小的数，比如 abs(a - b) < 1e-9。

认为变量有固定类型：在 Python 中，同一个变量可以先存整数再存字符串。虽然合法，但这会让代码难以阅读和维护。好的实践是保持变量类型一致，或者换一个有意义的变量名。
