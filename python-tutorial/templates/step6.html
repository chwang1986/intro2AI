文件与异常

让程序与外部世界安全地交互

场景程序不只是计算器

想象你正在写一份报告：你需要从文件夹里读取参考资料，写完后保存到硬盘。如果笔没墨了、文件被误删了，你不能直接崩溃大哭，而得优雅地处理这些意外。

程序也一样：它需要读写文件来持久化数据，也需要在出错时不至于"炸掉"，而是给用户一个友好的提示。文件操作和异常处理就是解决这两个问题的关键技能。

原理文件指针与异常传播链

打开文件时，操作系统会返回一个文件指针，像书签一样标记当前读写到哪个位置。read() 从头读到尾，write() 从当前位置写入。用完文件必须关闭，否则数据可能丢失，内存也会泄漏。

异常是程序运行时的"警报"。当错误发生时，Python 会创建一个异常对象并沿着调用链向上抛，直到被某个 try/except 捕获，否则会终止程序。finally 块则保证无论是否出错，清理代码都会执行。

图解

要点文件操作与异常处理

open() 函数接收文件名和模式（r 读/w 写/a 追加）；with 语句是最安全的文件操作方式，自动处理关闭。try/except/else/finally 构成完整的异常处理结构：try 放可能出错的代码，except 捕获特定异常，else 放成功后的逻辑，finally 放必须执行的清理。

代码示例
# 1. 基本文件读写（不推荐，容易忘记关闭）
f = open("data.txt", "r", encoding="utf-8")
content = f.read()
print(content)
f.close()   # 千万别忘！


# 2. with 语句（推荐）：自动关闭文件
with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()
    # 离开 with 块，文件自动关闭，即使出错也会关闭


# 3. 逐行读取（适合大文件，不占内存）
with open("data.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())   # strip() 去掉换行符


# 4. 写入文件
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("第一行\n")
    f.write("第二行\n")

# 追加模式
with open("output.txt", "a", encoding="utf-8") as f:
    f.write("第三行（追加）\n")


# 5. 完整的异常处理结构
try:
    with open("config.json", "r", encoding="utf-8") as f:
        data = f.read()
        number = int(data)   # 可能抛出 ValueError
except FileNotFoundError:
    print("错误：文件不存在，请检查路径")
except ValueError:
    print("错误：文件内容不是有效的数字")
except Exception as e:
    print(f"未知错误：{e}")
else:
    print(f"读取成功，数值为：{number}")   # 没异常时执行
finally:
    print("清理完成（无论是否出错都会执行）")

错误常见陷阱

1. 忘记关闭文件：打开文件不关闭会导致资源泄漏，在 Windows 上还可能锁定文件，其他程序无法访问。

# 错误
f = open("data.txt", "r")
data = f.read()
# 没有 f.close()！

# 正确：使用 with 语句
with open("data.txt", "r") as f:
    data = f.read()

2. 捕获所有异常却不处理：裸 except 会吞掉所有错误，包括系统退出信号，导致程序"死机"式卡死。

# 错误：捕获所有异常但什么都不做
try:
    risky_operation()
except:           # 不要这样写！
    pass          # 静默吞掉所有错误，调试时想死

# 正确：捕获具体异常并记录
except FileNotFoundError as e:
    print(f"文件未找到：{e}")
    raise           # 或重新抛出，让上层处理

3. 用 except 吞掉关键错误：不要在不理解错误的情况下静默处理，至少记录日志或重新抛出。

# 危险：让程序带着错误数据继续运行
try:
    result = calculate()
except Exception:
    result = 0   # 错误的默认值可能导致更大灾难

# 正确：要么处理，要么抛出
except ZeroDivisionError:
    result = float('inf')   # 合理的回退值
    logging.warning("除零，已设为正无穷")
