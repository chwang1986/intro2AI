01 Python 是什么

解释器工作原理，安装与运行第一个程序

场景为什么要学 Python

想象你想让电脑帮你做一件事——比如分析一份销售数据、自动下载图片、或者和 AI 对话。你需要用一种语言告诉电脑你的意图，而 Python 就是这种语言中最平易近人的一种。

在 AI 时代，Python 更像是一把万能钥匙。几乎所有主流的 AI 框架（TensorFlow、PyTorch）都用 Python 编写，各大 AI 公司的 API 也优先提供 Python SDK。学会 Python，就等于拿到了进入 AI 世界的门票。

与其他编程语言相比，Python 的语法更接近自然语言。比如用中文说「打印你好」，Python 代码就是 print("你好")，几乎一模一样。这种简洁让你能把注意力放在解决问题上，而不是纠结于复杂的语法规则。

起源Python 的诞生故事

1989 年的圣诞节，荷兰计算机研究员 Guido van Rossum（吉多·范罗苏姆）正在家里打发假期。他在荷兰国家数学和计算机科学研究所工作，之前参与过一门叫 ABC 的教学语言开发。ABC 的设计目标很美好——让非程序员也能轻松学习编程——但它过于理想化，连引号规则都很古怪，最终没能流行起来。

Guido 想：「能不能设计一门像 ABC 一样简洁优雅，又像 C 语言一样强大的语言？」于是他开始在圣诞假期里动手写代码。1991 年，第一个 Python 版本（0.9.0）发布到互联网上。这个名字来自他最喜欢的喜剧团体 Monty Python（蒙提·派森），和「蟒蛇」其实没有直接关系——虽然后来大家都用蛇作为标志。

Python 的初心是让人把时间花在解决问题上，而不是和机器斗智斗勇。Guido 有一句名言：「代码被阅读的次数远多于被编写的次数。」所以 Python 从诞生起就把「可读性」放在了第一位。

思考为什么需要解释型语言

在 Guido 写 Python 之前，编程对新手来说门槛极高。C 语言需要理解内存地址、指针、编译链接；Perl 被戏称为「只写语言」，三个月后连自己写的代码都看不懂。当时的程序员面临一个两难选择：

选择一：编译型语言（如 C/C++）——性能极高，但开发周期长。你写好代码后，要经过编译、链接才能运行。改一行代码，等编译就要几分钟。更麻烦的是，一个分号写错，满屏都是报错。

选择二：脚本语言（如 Bash、Perl）——写起来快，但语法晦涩难懂，不利于团队协作。

Guido 的思考是：能不能有一种语言，写起来像脚本一样快，读起来像伪代码一样清晰，同时还足够强大去做系统级的事情？ 这就是 Python 的核心定位。它选择了一条中间路线——解释执行，牺牲一部分运行速度，换取开发效率和代码可读性。

原理解释器就像一位翻译官

电脑只认识由 0 和 1 组成的机器语言，看不懂 Python 代码。所以 Python 需要一个「翻译官」——也就是解释器。解释器的工作分为三步：

1. 读取源码：解释器先读取你写的 .py 文件，检查语法是否正确。就像老师先通读你的作文，看看有没有错别字。

2. 编译成字节码：源码被转换成一种中间语言——字节码（bytecode）。这有点像把中文翻译成世界语，不是最终目标，但是一种标准化的中间形态。

3. 执行字节码：Python 虚拟机（PVM）逐条执行字节码指令，调用操作系统完成实际操作。这一步才是真正的「运行程序」。

整个过程是边解释边执行的。你每次运行 python hello.py，这个翻译过程都会重新进行。这正是 Python 被称为「解释型语言」的原因。

论证为什么解释执行是正确选择

你可能想问：既然编译型语言跑得更快，为什么 Python 还要选择解释执行？这其实是一个权衡问题——不同场景需要不同的最优解。

从开发效率看：写一个 Python 程序，从构思到运行可能只需要几分钟。而同样的功能用 C 写，光是处理内存分配和类型声明就要花很长时间。对于数据分析、人工智能原型开发、Web 后端这些场景，程序运行一次只需要几秒，但开发调试可能要几周——这时候开发效率比运行速度重要得多。

从可移植性看：编译型语言编译出来的可执行文件只能运行在特定操作系统上（Windows 的 .exe 不能在 Mac 上跑）。而 Python 源码可以在任何装了 Python 解释器的机器上运行——Windows、Mac、Linux，甚至树莓派。

从学习曲线看：解释型语言是「即时反馈」的。你写一行 print(1 + 2)，马上就能看到结果 3。这种即时感让初学者能快速建立信心，理解「程序在做什么」。

当然，如果你的目标是写一个每秒要处理百万请求的高频交易系统，Python 确实不是最佳选择——但即便如此，你也可以先用 Python 快速验证算法，再用 C++ 重写核心部分。很多 AI 框架（如 PyTorch）就是这样做的：Python 负责调用接口，底层 C++ 负责高速计算。

要点安装 Python，运行第一个程序

Python 有两个主要版本：Python 3 是目前的主流，Python 2 已经停止维护。本教程全部使用 Python 3。

安装完成后，打开终端输入 python --version，如果看到版本号就说明安装成功。然后创建一个名为 hello.py 的文件，写入一行代码，运行它。

代码示例
# 这是你的第一个 Python 程序
# 在终端运行: python hello.py

print("Hello, World!")
print("你好，Python！")

print() 是 Python 的内置函数，作用是把括号里的内容输出到屏幕。字符串需要用引号包裹，单引号 ' 和双引号 " 都可以。

实验亲手验证解释器的工作流程

光听原理不够，我们来亲手做几个实验，看看 Python 解释器到底是怎么工作的。

实验 1：观察交互式解释器

打开终端，输入 python3（Windows 上可能是 python），你会看到类似下面的提示符：

$ python3
Python 3.11.4 (main, Jun  9 2023, 07:23:44)
>>> print("我在交互式解释器里！")
我在交互式解释器里！
>>> 1 + 2
3
>>> exit()

注意：你每按一次回车，解释器就立刻编译并执行这一行——这就是「边解释边执行」。试着输入 1 / 0，观察报错信息，理解解释器是在运行时才发现错误的。

实验 2：查看字节码

Python 把源码编译成字节码后执行。我们可以用内置的 dis 模块来查看这段字节码长什么样：

import dis

def hello():
    print("Hello, World!")

dis.dis(hello)

# 输出中会看到类似 LOAD_GLOBAL、CALL_FUNCTION 这样的指令
# 这些就是 Python 虚拟机（PVM）真正执行的命令

实验 3：对比编译型与解释型的差异

创建一个 hello.py 文件，内容只有 print("Hello")。运行 python hello.py，你会发现在项目目录中生成了 __pycache__ 文件夹，里面有 .pyc 文件——这就是编译后的字节码缓存。下次再运行同一个文件时，Python 会跳过编译步骤，直接加载缓存的字节码，加快启动速度。

错误初学者常犯的错误

Python 2 和 3 混用：网上有些旧教程使用 Python 2 语法，比如 print "hello"（不带括号）。在 Python 3 中这会直接报错。记住：Python 3 的 print 必须带括号。

缩进混用空格和 Tab：Python 用缩进来表示代码层级，但对缩进方式很敏感。同一段代码里不能混用空格和 Tab，否则会出现 IndentationError。建议把编辑器设置成「按 Tab 键时插入 4 个空格」。

文件名带中文或特殊符号：虽然 Python 支持中文文件名，但在某些系统上可能引发编码问题。初学时建议用纯英文文件名，比如 hello.py 而不是 你好.py。
