AI 协作工作流

把 AI 变成你的「编程搭档」——而不是「代码打印机」

场景一个人写项目太慢，两个人又太贵

想象你要盖一座小木屋。你不会一口气画出整栋房子的图纸然后交给工人——而是先设计地基，等地基稳固了再搭框架，框架好了再上屋顶。每一步都检查、调整、确认。

用 AI 做编程项目也是如此。AI 协作工作流就是把一个复杂项目拆解成多个可管理的阶段，让 AI 在每个阶段扮演「超级助手」的角色：帮你生成初稿、提供思路、检查漏洞。但你始终是「项目经理」——审查、决策、把关质量。

本章将教你一套经过实战验证的协作流程，让 AI 真正成为你的「编程搭档」，而不是一个乱码生成器。

原理循环迭代：需求 → 生成 → 审查 → 测试 → 优化

高质量的 AI 协作不是「一次性交易」，而是一个持续迭代的循环。就像雕塑家先用 clay 捏出大致形状，再反复打磨细节。

工作流分为五个阶段：需求拆解（把大目标拆成小模块）、AI 生成（为每个模块获取初稿）、人工审查（检查逻辑、安全、风格）、测试验证（运行代码、检查边界情况）、迭代优化（把问题反馈给 AI 继续改进）。

这个循环会重复多次，直到每个模块都达到可交付标准。秘诀是：小步快跑，频繁验证。

图解

要点五个协作实战技巧

下面是让 AI 协作工作流高效运转的五个核心技巧：

1. 需求描述技巧：用用户故事（User Story）描述需求。"作为一个用户，我希望能够按截止日期排序任务，以便优先处理紧急事项。"这种格式天然适合拆解为函数和接口。

2. 分模块让 AI 实现：不要要求 AI 一次性生成整个项目。先让它设计数据模型，确认后再写 CRUD 操作，然后再写业务逻辑。每步独立验证。

3. 代码审查清单：建立自己的审查清单——是否处理了空值？是否有 SQL 注入风险？异常处理是否完整？命名是否清晰？像审查同事代码一样审查 AI 的代码。

4. 测试驱动：先让 AI 写测试用例，再让它写实现代码。测试就像「自动审查员」，能快速发现 AI 代码的边界问题。

5. 版本控制配合：每次让 AI 生成或修改代码后，及时 commit。这样当 AI「改坏」东西时，你可以轻松回滚。把对话历史也保存下来，方便追溯思路。

代码示例
"""
AI 协作工作流实战示例：用 AI 协助开发一个「任务管理器」模块

步骤1：让 AI 设计数据模型
Prompt: "你是一位 Python 架构师。请设计一个 Task 数据类，
包含 id、title、description、priority、due_date、completed 字段。
使用 dataclass，添加类型注解，priority 用 Enum 约束为 LOW/MEDIUM/HIGH。"

步骤2：让 AI 写单元测试
Prompt: "基于上面的 Task 类，请用 pytest 写一组单元测试，
验证：创建任务、标记完成、优先级排序。至少覆盖 5 个测试用例。"

步骤3：让 AI 实现 CRUD 操作  
Prompt: "请实现一个 TaskManager 类，提供以下方法：
- add_task(task: Task) -> None
- get_task(task_id: int) -> Task | None
- list_tasks(completed_only: bool = False) -> list[Task]
- complete_task(task_id: int) -> bool
- delete_task(task_id: int) -> bool
使用内存存储（list），添加适当的错误处理。"

步骤4：人工审查并整合
- 检查是否有线程安全问题
- 检查 ID 自增逻辑是否正确
- 检查异常类型是否恰当
- 统一代码风格，添加文档字符串

步骤5：运行测试并迭代
"""

import pytest
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from typing import Optional


class Priority(Enum):
    LOW = 1
    MEDIUM = 2
    HIGH = 3


@dataclass
class Task:
    """任务数据模型"""
    id: int
    title: str
    description: str = ""
    priority: Priority = Priority.MEDIUM
    due_date: Optional[datetime] = None
    completed: bool = False
    created_at: datetime = field(default_factory=datetime.now)


class TaskManager:
    """任务管理器：经过人工审查和优化后的版本"""
    
    def __init__(self):
        self._tasks: list[Task] = []
        self._next_id: int = 1
    
    def add_task(self, title: str, description: str = "",
                 priority: Priority = Priority.MEDIUM,
                 due_date: Optional[datetime] = None) -> Task:
        """添加新任务，自动分配 ID"""
        if not title or not title.strip():
            raise ValueError("任务标题不能为空")
        
        task = Task(
            id=self._next_id,
            title=title.strip(),
            description=description.strip(),
            priority=priority,
            due_date=due_date
        )
        self._tasks.append(task)
        self._next_id += 1
        return task
    
    def get_task(self, task_id: int) -> Optional[Task]:
        """根据 ID 获取任务"""
        for task in self._tasks:
            if task.id == task_id:
                return task
        return None
    
    def list_tasks(self, completed_only: bool = False,
                   sort_by_priority: bool = True) -> list[Task]:
        """列出任务，支持过滤和排序"""
        result = [t for t in self._tasks
                  if not completed_only or t.completed]
        if sort_by_priority:
            result.sort(key=lambda t: t.priority.value, reverse=True)
        return result
    
    def complete_task(self, task_id: int) -> bool:
        """标记任务为已完成"""
        task = self.get_task(task_id)
        if task and not task.completed:
            task.completed = True
            return True
        return False
    
    def delete_task(self, task_id: int) -> bool:
        """删除任务"""
        task = self.get_task(task_id)
        if task:
            self._tasks.remove(task)
            return True
        return False


# 测试驱动验证
class TestTaskManager:
    def test_add_task(self):
        tm = TaskManager()
        task = tm.add_task("学习 Python", priority=Priority.HIGH)
        assert task.id == 1
        assert task.title == "学习 Python"
        assert task.priority == Priority.HIGH
    
    def test_empty_title_raises_error(self):
        tm = TaskManager()
        with pytest.raises(ValueError, match="任务标题不能为空"):
            tm.add_task("")
    
    def test_complete_task(self):
        tm = TaskManager()
        task = tm.add_task("阅读文档")
        assert tm.complete_task(task.id) is True
        assert task.completed is True
        assert tm.complete_task(task.id) is False  # 已完成的不能再完成
    
    def test_list_tasks_sorted_by_priority(self):
        tm = TaskManager()
        tm.add_task("低优先级", priority=Priority.LOW)
        tm.add_task("高优先级", priority=Priority.HIGH)
        tm.add_task("中优先级", priority=Priority.MEDIUM)
        tasks = tm.list_tasks()
        assert tasks[0].priority == Priority.HIGH
        assert tasks[1].priority == Priority.MEDIUM
        assert tasks[2].priority == Priority.LOW


if __name__ == "__main__":
    pytest.main([__file__, "-v"])

错误协作工作流的四大禁忌

AI 协作工作流最大的敌人不是 AI 不够聪明，而是人类太懒惰或太贪心：

1. 让 AI 一次生成整个项目 —— 这就像让一位厨师同时做前菜、主菜、甜点，还要求一次上桌。结果必然是每道菜都马马虎虎。模块化管理，逐个击破。

2. 不审查直接复制 —— AI 会一本正经地生成有安全漏洞的代码。SQL 注入、路径遍历、敏感信息泄露……这些风险不会因为你「信任」AI 就消失。把 AI 代码当作「实习生提交的 PR」，逐行审查。

3. 不测试就上线 —— AI 写的代码可能在 90% 的情况下运行正确，但在边界条件下崩溃。空列表、超大输入、特殊字符——这些都是 AI 的盲区。测试覆盖率是 AI 协作项目的生命线。

4. 和 AI 对话没有保存历史 —— 你花了半小时和 AI 反复打磨出一个完美函数，结果手滑刷新了页面，所有上下文丢失。养成保存对话、导出代码、及时 commit 的习惯。用版本控制记录每一次 AI 协助的修改。
