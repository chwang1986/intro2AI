数据结构

用合适的容器高效组织数据

起源从数组到哈希表的进化之路

计算机刚诞生时，程序只能操作单个数字。1950 年代，Fortran 语言引入了数组（Array）——把一批同类型的数据连续存放在内存中，用索引访问。数组简单高效，但有两个硬伤：大小固定（不能随意增删），查找元素需要逐个扫描。

1960 年代，计算机科学家发明了链表（Linked List）：每个数据节点除了存储值，还保存一个指向下一个节点的「指针」。这解决了数组插入删除困难的问题，但牺牲了随机访问能力——要找第 100 个元素，必须从第 1 个开始逐个跳过去。

1970 年代，一项革命性的发明出现了——哈希表（Hash Table）。它的核心思想是：通过一个「哈希函数」把键（key）直接映射到内存地址，实现近乎瞬时的查找。无论表里有 10 个还是 10 万个元素，查找时间都差不多。Python 的字典（dict）就是基于哈希表实现的。1980 年代以后，几乎所有现代编程语言都把哈希表作为内置类型，因为它实在太实用了。

至于集合（Set），它源于数学中的集合论。19 世纪的数学家康托尔奠定了集合论基础，而计算机科学家后来发现：如果用哈希表只存储「键」而不存储「值」，就得到了一个自动去重、查找极快的集合结构。

思考算法 + 数据结构 = 程序

计算机科学家 Niklaus Wirth（Pascal 语言之父）写过一本经典教材，书名就叫《Algorithms + Data Structures = Programs》（算法 + 数据结构 = 程序）。这个等式揭示了一个核心道理：选择什么样的容器，决定了你能用什么样的算法，进而决定了程序的效率。

举个例子：假设你有一百万个用户的手机号，需要频繁判断「某个手机号是否在列表中」。如果你用列表存储，每次查找平均要扫描五十万个元素。如果你用集合存储，借助哈希表，每次查找只需要计算一次哈希值，瞬间就能知道结果——速度提升了上万倍。

再举个例子：你要存储学生的成绩，并且需要按学号快速查到某个学生的分数。如果用列表，你需要遍历整个列表找匹配项；如果用字典，直接把学号作为键、成绩作为值，一次查找就能定位。这就是为什么字典被称为 Python 中最强大的数据结构之一。

Python 的设计者 Guido 在创造这门语言时，刻意把这四种最常用的数据结构直接内置到语言核心中。你不需要像 C 语言那样自己实现链表或哈希表，开箱即用。这大大降低了编程门槛，也让 Python 成为处理数据的利器。

场景如何管理一堆数据？

想象你要整理家里的物品：列表像一排书架，书按顺序摆放，你可以随时抽出一本换掉；字典像带标签的收纳盒，写上"袜子"就能直接找到袜子，不用翻遍整个柜子；元组像被封死的展示柜，里面的珍品只能看不能碰；集合像抽奖箱，东西在里面乱滚，但绝不会出现两个一模一样的球。

不同场景需要不同的容器——选对了，代码既简洁又高效。

原理四种容器的特性对比

Python 提供了四种内置容器，核心差异在于是否有序、是否可变、是否允许重复。理解这三点，就能在正确的场景选对容器。

图解

要点四种容器的使用

列表是最通用的"篮子"，适合需要顺序和修改的场景；字典通过键值对实现 O(1) 查找，是构建映射关系的首选；元组轻量且不可变，适合作为字典的键或函数的多返回值；集合自动去重，成员检测极快，适合交集并集运算。

代码示例
# 1. 列表 list：有序可变，用 [] 定义
fruits = ["苹果", "香蕉", "橙子"]
fruits.append("葡萄")          # 末尾添加
fruits.insert(1, "芒果")       # 在索引 1 插入
fruits[0] = "西瓜"             # 修改
print(fruits)                  # ['西瓜', '芒果', '香蕉', '橙子', '葡萄']
print(fruits[1:3])             # 切片：['芒果', '香蕉']


# 2. 字典 dict：键值对映射，用 {} 定义
user = {"name": "小明", "age": 18, "city": "北京"}
print(user["name"])            # 小明
user["age"] = 19               # 修改值
user["gender"] = "男"          # 新增键值对

# 安全取值：get 方法不会报错
print(user.get("phone", "未知"))   # 未知

# 遍历字典
for key, value in user.items():
    print(f"{key}: {value}")


# 3. 元组 tuple：有序不可变，用 () 定义
point = (10, 20)
x, y = point                   # 解包
print(x, y)                    # 10 20

# 元组可以作为字典的键（因为不可变）
locations = {
    (0, 0): "原点",
    (10, 20): "目标点"
}


# 4. 集合 set：无序不重复，用 set() 或 {} 定义（空集合必须用 set()）
a = {1, 2, 3, 3, 3}
print(a)                       # {1, 2, 3} —— 自动去重

b = {3, 4, 5}
print(a & b)                   # 交集: {3}
print(a | b)                   # 并集: {1, 2, 3, 4, 5}
print(a - b)                   # 差集: {1, 2}
print(2 in a)                  # 成员检测: True（极快）

论证为什么 Python 内建这四种容器

你可能会问：为什么 Python 不提供「链表」「栈」「队列」这些经典数据结构，而是选择了列表、字典、元组、集合这四种？这其实是一个经过深思熟虑的设计决策——用少数几个强大的通用结构，覆盖绝大多数编程场景。

列表 = 动态数组：Python 的列表底层不是纯链表，而是「动态数组」。它兼顾了数组的随机访问速度（通过索引瞬间定位）和链表的动态扩容能力。对于大多数日常任务——存储序列、遍历、追加——列表已经足够高效。只有当你需要在列表头部频繁插入删除时，才需要考虑 collections.deque（双端队列）。

字典 = 哈希表：字典是 Python 最核心、最高频的数据结构。它用空间换时间，实现了 O(1) 的查找速度。从 Python 3.7 开始，字典还保证了插入顺序——这是哈希表实现上的一项重大优化，让字典同时兼具了映射查找和有序存储的能力。

元组 = 轻量不可变记录：元组存在的意义不是「不能改的列表」，而是「轻量级的结构化记录」。比如坐标 (x, y)、RGB 颜色 (255, 128, 0)，这些数据天然应该作为一个整体被传递，且不应该被意外修改。元组的不可变性还让它可以作为字典的键——这是列表做不到的。

集合 = 去重与关系运算：如果你只需要判断「某个元素是否存在」，集合比字典更省空间（不需要存储值）；如果你需要做交集、并集、差集运算，集合提供了直观运算符 & | -。在数据清洗中，集合的去重功能几乎是标配。

对比 C++ 的 STL（标准模板库）或 Java 的 Collections 框架——它们提供了几十种容器，每一种都有特定的适用场景，但学习曲线非常陡峭。Python 的哲学是「一种明显的方式去做一件事」，四种内置容器覆盖了 90% 的使用场景，剩下的 10% 可以通过标准库的 collections 模块补充。

实验亲手感受不同容器的特性

实验 1：验证列表和字典的查找速度差异

创建一个包含 10000 个元素的列表和字典，分别在其中查找目标元素，用 time 模块测量耗时：

import time

# 准备数据
n = 100000
my_list = list(range(n))
my_dict = {i: i for i in range(n)}
target = n - 1

# 列表查找
start = time.time()
found = target in my_list
print("列表查找耗时:", time.time() - start)

# 字典查找
start = time.time()
found = target in my_dict
print("字典查找耗时:", time.time() - start)

# 你会看到字典查找快得多，即使数据量达到百万级也几乎瞬间完成

实验 2：观察元组的不可变性和「内部可变」现象

运行下面的代码，理解「元组本身不可变」和「元组内部的可变对象可变」之间的区别：

t = (1, 2, [3, 4])
print("原始元组:", t)

# t[0] = 10       # 报错！元组元素不能被重新赋值
t[2].append(5)   # 成功！因为列表本身是可变的
print("修改后:", t)

t[2] = [6, 7]    # 报错！虽然列表内容可变，但不能把整个列表换成另一个

实验 3：体验集合的去重和运算

假设你有两个班级的学生名单，想找出「同时参加两个班」和「只参加一个班」的学生。用集合运算可以非常优雅地解决：

class_a = {"小明", "小红", "小刚", "小丽"}
class_b = {"小红", "小丽", "小华", "小杰"}

print("两个班都参加的:", class_a & class_b)       # 交集
print("至少参加一个班的:", class_a | class_b)      # 并集
print("只参加 A 班的:", class_a - class_b)         # 差集
print("只参加一个班的:", class_a ^ class_b)        # 对称差集

# 用列表做同样的运算会复杂很多——你需要写循环逐个比较

错误常见陷阱

1. 用可变对象做 dict 的 key：字典的键必须是不可变类型（可哈希），列表和字典不能当键。

# 错误
my_dict = {}
key = [1, 2]
my_dict[key] = "value"   # TypeError: unhashable type: 'list'

# 正确：改用元组
key = (1, 2)
my_dict[key] = "value"   # 正常

2. 修改 tuple 元素：元组创建后不能修改，但元组内的可变元素（如列表）可以被修改。

t = (1, 2, [3, 4])
# t[0] = 10   # TypeError: 'tuple' object does not support item assignment

t[2].append(5)   # 这是可以的！因为列表本身是可变的
print(t)         # (1, 2, [3, 4, 5])

3. 列表遍历时删除元素：在遍历过程中删除会导致索引错乱，跳过某些元素。

# 错误示范
nums = [1, 2, 3, 4, 5]
for i, n in enumerate(nums):
    if n % 2 == 0:
        del nums[i]   # 危险！删除后后续元素前移，会跳过某些元素

# 正确做法：用列表推导式或倒序删除
nums = [1, 2, 3, 4, 5]
nums = [n for n in nums if n % 2 != 0]   # 保留奇数: [1, 3, 5]
