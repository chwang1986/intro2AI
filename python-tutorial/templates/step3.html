03 控制流

条件判断与循环，让程序拥有决策能力

场景程序需要做出选择和重复

想象你每天早上出门前的决策过程：如果下雨，就带伞；如果温度低于 10 度，就穿羽绒服；否则穿普通外套。这不是直线式的「做 A，然后做 B」，而是根据条件选择不同的路径。

程序也需要这种能力。一个购物网站要判断「用户是否登录」来决定显示登录页还是个人中心；一个数据处理脚本要「重复读取每一行」直到文件结束。没有控制流，程序就只能从上到下死板地执行，和计算器没什么区别。

控制流让程序拥有了「智商」——它能根据情况做判断，能重复执行任务，能跳过不必要的步骤。这是从「脚本」走向「程序」的关键一跃。

起源从「面条代码」到结构化程序

在计算机发展的早期，程序的控制流全靠一条叫 GOTO 的指令实现。它可以让程序跳转到任意位置，就像在一本书里随意翻页。1968 年，计算机科学家 Edsger Dijkstra（艾兹格·迪科斯彻）发表了一封著名的信《Go To Statement Considered Harmful》（GOTO 语句有害论），指出滥用 GOTO 会让程序变成一团纠缠不清的「面条代码」（Spaghetti Code），没人能看懂，更没人敢修改。

Dijkstra 的呼声引发了长达数年的争论。1966 年，意大利数学家 Corrado Böhm 和 Giuseppe Jacopini 其实已经用数学证明了：任何程序都可以用三种基本结构组合而成——顺序、分支（选择）和循环。这个结论被称为「结构化程序定理」。既然三种结构就够了，为什么还要用危险的 GOTO 呢？

到了 1970 年代，Pascal、C 等新一代语言纷纷抛弃了 GOTO，拥抱结构化编程。Python 作为一门现代语言，从设计之初就完全遵循了这一思想：没有 GOTO，只有 if（分支）和 for/while（循环）。这让 Python 代码天然具有清晰的逻辑层次，读起来像一篇结构良好的文章。

思考为什么程序不能只有「从上到下」

假设你正在写一个简单的「用户登录验证」程序。如果没有控制流，代码只能从上到下直线执行：询问用户名 → 询问密码 → 显示登录成功。问题是——如果密码错了怎么办？如果没有控制流，你没有任何办法说「如果密码错误，提示重新输入」。程序就像一个只会按固定路线行驶的火车，遇到障碍也不会转弯。

现实世界充满了选择和重复：如果你考了 60 分以上就及格，否则要补考；每天都要背单词，直到背完一整本；对表格的每一行数据做同样的计算……这些场景分别对应了三种基本结构：

分支（if）：处理「二选一」或「多选一」的场景。就像路口的交通灯，红灯停、绿灯行，条件不同，路线不同。

循环（for/while）：处理「重复做某事」的场景。for 更像「遍历已知的清单」（如「对班里每个同学点名」），while 更像「直到满足某个条件为止」（如「一直练钢琴，直到弹对为止」）。

Python 的设计者 Guido 在选择循环语法时，刻意避开了 C 语言那种复杂的 for (i=0; i<n; i++) 写法，而是选择了更直观的 for item in items——「对于列表中的每个元素，做某事」。这种设计让代码读起来像英语句子，大大降低了理解门槛。

原理程序执行的三种基本结构

无论多么复杂的程序，其控制流都可以分解为三种基本结构的组合：顺序、分支和循环。这是计算机科学家早在 1966 年就证明的定理——结构化程序定理。

顺序结构：代码从上到下依次执行，就像按照菜谱一步一步做菜。这是默认的执行方式。

分支结构（if）：根据条件选择执行路径，就像走到岔路口看路牌决定往哪边走。Python 用 if、elif、else 实现分支。

循环结构（for / while）：重复执行某段代码，就像洗衣机按照设定程序反复转动。Python 提供了 for（遍历序列）和 while（条件为真时持续执行）两种循环。

论证三种结构为什么足够表达任何逻辑

Böhm 和 Jacopini 在 1966 年的证明听起来很抽象，我们来用一个具体例子理解它。假设你要写一个「自动售货机」程序，流程如下：先显示商品列表（顺序），然后根据用户选择出货（分支），如果用户投入金额不足就提示并等待再次投币（循环）。这个程序里有没有 GOTO？没有。但有没有我们表达不了的逻辑？也没有。

关键洞察是：任何复杂的逻辑都可以拆解为「先做 A，然后根据 B 决定做 C 还是 D，重复 E 直到 F」这样的组合。就像乐高积木只有几种基本形状，但你可以搭出城堡、飞船、摩天大楼。程序的结构化思维也是如此——掌握三种基本结构，然后通过嵌套和组合，就能表达任意复杂的算法。

对比 GOTO 的「自由跳转」，结构化编程的代价是什么？有时候代码会稍微长一点——比如从深层嵌套中提前退出，用 GOTO 只需要一行，用结构化语法可能需要设置额外的标志变量或者用 break。但换来的好处是巨大的：代码的逻辑路径是「可预测的」，你不需要在脑子里维护一张「跳转地图」，只需要顺着缩进层级往下读，就能理解程序的行为。对于团队协作和代码维护来说，这种可读性的提升远超过了那点额外的代码量。

Python 甚至走得更远：它强制要求统一的缩进，从语法层面保证了代码的结构清晰。在 C 语言里，缩进只是「风格建议」；在 Python 里，缩进就是语法本身。这让 Python 代码在视觉上天然具有结构感，不会出现「缩进和大括号不匹配」的混乱情况。

要点条件判断与循环

if 语句让程序做选择，for 和 while 让程序重复执行。break 可以立即跳出循环，continue 则跳过当前轮次直接进入下一轮。

Python 的 for 循环设计得非常直观：「对于列表中的每一个元素，做某事」。你不需要关心索引和边界，只需要描述「对每个 item 做什么」。这让代码读起来像英文句子一样自然。

代码示例
# if / elif / else 条件判断
score = 85

if score >= 90:
    print("优秀")
elif score >= 60:
    print("及格")      # 这一行会被执行
else:
    print("不及格")

# for 循环遍历列表
fruits = ["苹果", "香蕉", "橙子"]
for fruit in fruits:
    print(fruit)

# while 循环
n = 3
while n > 0:
    print(n)
    n -= 1             # 不要忘记更新条件变量！

# break 和 continue
for num in range(10):
    if num == 3:
        continue       # 跳过 3，继续下一轮
    if num == 7:
        break          # 遇到 7，直接结束循环
    print(num)         # 输出: 0, 1, 2, 4, 5, 6

实验亲手观察控制流的执行轨迹

实验 1：用 print() 追踪程序的执行路径

在代码的关键位置插入 print()，观察程序实际走了哪条路径、循环执行了几次。这是理解控制流最直观的方法：

score = 75
print("开始判断...")

if score >= 90:
    print("进入 >=90 分支")
elif score >= 60:
    print("进入 >=60 分支")   # 你会看到这一行被打印
else:
    print("进入 else 分支")

print("判断结束。")

# 循环追踪
for i in range(3):
    print(f"第 {i} 次循环开始")
    if i == 1:
        print("  遇到 i==1，跳过本次剩余部分")
        continue
    print(f"  处理 i={i}")
    print(f"第 {i} 次循环结束")

运行上面的代码，观察输出顺序。特别注意 continue 的作用：当 i == 1 时，「处理 i=1」和「第 1 次循环结束」这两行没有被打印，因为 continue 直接跳到了下一次循环的开头。

实验 2：故意制造一个死循环并观察

在交互式解释器中输入下面的代码（注意：准备好按 Ctrl+C 终止！）：

>>> n = 3
>>> while n > 0:
...     print(n)
...
3
3
3
3
...（无限输出）...
# 按 Ctrl+C 终止
KeyboardInterrupt

为什么死循环了？因为 n 永远是 3，条件 n > 0 永远为真。修复方法是在循环体内加入 n -= 1。这个实验让你切身体会到：写 while 循环时，必须确保循环变量会向终止条件靠近。

实验 3：对比 for 和 while 的适用场景

试着用两种循环实现同一个功能：计算 1 到 10 的和。观察哪种写法更自然、更不容易出错：

# 用 for 实现
 total = 0
 for i in range(1, 11):
     total += i
 print(total)   # 55

 # 用 while 实现
 total = 0
 i = 1
 while i <= 10:
     total += i
     i += 1     # 如果忘了这行，就是死循环！
 print(total)   # 55

体会一下：for 循环自动帮你管理计数器，不容易出错；while 循环更灵活，但需要手动维护循环变量。在「遍历已知范围」时，优先使用 for；在「条件驱动」时，才使用 while。

错误初学者常犯的错误

忘记缩进：Python 用缩进来表示代码块，而不是大括号。if 体、循环体、函数体都必须有统一的缩进。一旦混用空格和 Tab，或者某一行少缩进了一格，就会抛出 IndentationError。

while 死循环：如果 while 的条件永远为真，循环就永远不会结束。比如 while n > 0: print(n) 却忘了写 n -= 1，程序会疯狂打印 3 直到你强制终止。写 while 循环时，一定要确保条件最终会变为假。

在循环中修改正在遍历的列表：下面这段代码的预期是删除所有偶数，但结果却是错的——因为删除元素后列表长度和索引都发生了变化，循环会跳过某些元素。正确做法是遍历副本 for num in nums[:]:，或者先收集要删除的元素。

错误示例
# 错误：在遍历列表时直接删除元素
nums = [1, 2, 3, 4]
for num in nums:
    if num % 2 == 0:
        nums.remove(num)   # 危险！会跳过元素
print(nums)                # 结果可能是 [1, 3] 或更奇怪的东西

# 正确：遍历列表的副本
nums = [1, 2, 3, 4]
for num in nums[:]:
    if num % 2 == 0:
        nums.remove(num)
print(nums)                # [1, 3]
