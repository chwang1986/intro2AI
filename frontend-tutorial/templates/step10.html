Step 10 页面模板

可直接套用的页面结构与实战组合

实战

场景

学完前面的设计令牌、组件和布局，现在需要把它们组合成完整的页面。页面模板就是"现成的配方"，告诉你在什么场景下用什么组件、怎么排列。以后做新页面时，直接选一个最接近的模板，复制粘贴再微调。

原理

原子设计方法论（Atomic Design）

Brad Frost 提出的原子设计，把界面分成五个层级：

  • 原子（Atoms）：最小的元素，如按钮、输入框、标签。对应我们的 Step 5 基础组件。
  • 分子（Molecules）：原子组合成的小组件，如搜索框（输入框 + 按钮）。
  • 有机体（Organisms）：分子组合成的区块，如导航栏、卡片列表。对应我们的 Step 6 容器组件和 Step 7 数据展示。
  • 模板（Templates）：有机体组合成的页面骨架，没有真实内容。对应本章。
  • 页面（Pages）：模板填充真实内容后的最终页面。

这套方法论的核心是自下而上地构建：先定义原子，再组合成分子、有机体，最后搭成页面。这样任何一个层级的改动都能自动同步到所有使用它的地方。

要点

  1. 每个页面有明确的"一页一事"：一个页面只做一件事（展示数据、填写表单、阅读文档）。不要把所有功能塞进一个页面。
  2. 页面结构保持一致：Header + Main + Footer 的大框架不变，具体内容在变。用户打开新页面时，不需要重新学习布局。
  3. 优先展示核心内容：页面加载后，用户第一眼应当看到最重要的信息，而不是被广告、提示、弹窗干扰。
  4. 面包屑用于深层级：如果页面嵌套超过两层（如 首页 / 教程 / 前端设计 / 色彩系统），用面包屑告诉用户当前位置。
  5. 预留扩展空间：模板不要写得太死，给未来可能增加的内容留位置。

常见错误

❌ 每页结构都不一样

这个页面有侧边栏，那个没有，另一个侧边栏在右边。用户需要不断适应新布局，增加认知负担。

❌ 模板写得太死

模板里所有尺寸、颜色都写死 px，复用时只能全盘复制，无法微调。应当用 CSS 变量和 class，方便覆盖。

❌ 忽略无数据状态

模板里放满了示例数据，但没有考虑真实场景中数据为空、加载失败、权限不足的情况。

练习

练习 1：用本教程的组件类，搭建一个"个人博客文章页"模板，包含文章标题、作者信息、正文、评论区。

练习 2：为 Git 教程或 Python 教程设计一个新的首页模板，使用本教程的卡片、按钮、导航组件。

练习 3：把本教程的所有页面模板整理成一个 HTML 文件，作为"快速启动模板"，以后新项目直接复制。
