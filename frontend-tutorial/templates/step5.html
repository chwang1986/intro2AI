Step 5 基础组件

按钮、输入框、标签、徽章的规范与变体

组件层

场景

页面上最常见的交互元素就是按钮和输入框。如果没有统一规范，每个开发者写出来的按钮大小、颜色、圆角都不一样，用户会觉得这个产品在"拼凑"。基础组件的目标是把最常用的元素标准化，让所有人复制同一套 class 就能用。

原理

Ant Design 的组件设计思路

Ant Design 把按钮分成五种类型（主按钮、次按钮、虚线按钮、文字按钮、链接按钮），每种类型对应不同的使用场景：

  • 主按钮（Primary）：一个页面上只有一到两个，用于最重要的操作（提交、保存）。
  • 次按钮（Default）：最常用的按钮，用于普通操作（取消、返回）。
  • 虚线按钮（Dashed）：用于添加、导入等次要操作，视觉上比次按钮更弱。
  • 文字按钮（Text）：用于不重要的操作，或者空间有限的场景。
  • 链接按钮（Link）：用于跳转外部页面或打开新窗口。

输入框的设计原则

  • 高度统一：标准 36px，大 44px，小 28px。和按钮高度对齐，方便并排。
  • 聚焦状态要明显：边框变色 + 外发光，让用户知道当前在哪个输入框。
  • placeholder 只是辅助：不能把 placeholder 当成 label 用，因为用户输入后它就消失了。

要点

  1. 按钮类型按重要性递减：Primary → Ghost → Dashed → Text → Link。一个页面上 Primary 按钮不超过两个。
  2. 危险操作用红色：删除、清空、退出登录等不可逆操作，用 btn-danger 给用户心理预警。
  3. 标签（Tag）用颜色表达语义：蓝色=进行中，绿色=成功，黄色=警告，红色=失败。不要让用户去猜。
  4. 徽章（Badge）只用于数字提示：消息未读数、待处理任务数。不要用它来装饰。
  5. 禁用状态要一眼看出：降低透明度到 50%，去掉 hover 效果，鼠标变成 not-allowed。

常见错误

❌ 按钮主次不分

页面上有三个主按钮（蓝色），用户不知道哪个是主要操作。应当只有一个 Primary，其余用 Default 或 Text。

❌ 输入框没有聚焦样式

用户点了输入框，但没有任何视觉反馈，不知道自己有没有选中。必须加边框变色和外发光。

❌ 标签颜色与语义不符

用红色标签表示"进行中"，绿色标签表示"已删除"。颜色有强烈的心理暗示，不能乱用。

完整代码

<!-- 按钮 -->
<button class="btn btn-primary">主按钮</button>
<button class="btn btn-ghost">次按钮</button>
<button class="btn btn-dashed">虚线按钮</button>
<button class="btn btn-text">文字按钮</button>
<button class="btn btn-link">链接按钮</button>
<button class="btn btn-danger">危险按钮</button>

<!-- 输入框 -->
<input class="input" placeholder="标准输入框">
<input class="input input-lg" placeholder="大号输入框">
<input class="input input-sm" placeholder="小号输入框">
<textarea class="textarea" placeholder="文本域"></textarea>

<!-- 标签 -->
<span class="tag tag-brand">进行中</span>
<span class="tag tag-success">成功</span>
<span class="tag tag-warning">警告</span>
<span class="tag tag-error">失败</span>

练习

练习 1：创建一个表单页面，包含用户名（输入框）、密码（输入框）、提交（主按钮）、重置（次按钮）。确保按钮和输入框高度对齐。

练习 2：为一个任务列表页面设计标签系统，包含"待处理""进行中""已完成""已取消"四种状态，每种状态用对应的颜色标签。

练习 3：给上面的表单加上禁用状态（提交按钮在输入为空时禁用），测试禁用样式的视觉效果是否明显。
