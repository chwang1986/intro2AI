Step 5 基础组件

按钮、输入框、标签、徽章的规范与变体

组件层

场景

页面上最常见的交互元素就是按钮和输入框。如果没有统一规范，每个开发者写出来的按钮大小、颜色、圆角都不一样，用户会觉得这个产品在"拼凑"。基础组件的目标是把最常用的元素标准化，让所有人复制同一套 class 就能用。

起源

在 GUI（图形用户界面）出现之前，人与计算机的交互完全依赖命令行（CLI）——用户输入文本命令，计算机返回文本结果。1981 年，Xerox Star 工作站首次引入了窗口、图标、菜单、指针（WIMP）的交互范式，按钮、输入框、复选框等组件由此诞生。

早期的 GUI 组件严格模拟物理世界：按钮有凸起的 3D 效果，按下去时凹陷，让用户感知到"这是一个可以被按下的东西"。这种设计被称为拟物化（Skeuomorphism）。2013 年，iOS 7 引领了扁平化设计革命，去掉了多余的装饰，用颜色、阴影、动效来传达交互状态。现代设计系统（Ant Design、Material Design）在此基础上进一步抽象，将组件定义为状态机：每个组件有一组明确的状态（默认、悬停、激活、禁用、加载），状态之间的转换规则是设计系统的核心。

组件化的概念也从设计领域延伸到工程领域。React、Vue 等前端框架的组件化思想，本质上是对设计组件的代码映射：一个按钮组件封装了 HTML 结构、CSS 样式和交互行为，开发者通过 props 传入参数（类型、尺寸、状态），即可获得一致的表现。

思考

为什么按钮是圆角而不是直角？

圆角按钮比直角按钮更"友好"，这不仅是审美偏好，还有认知心理学的解释。格式塔心理学的连续性原则指出，人眼更容易跟随平滑的曲线，而非尖锐的折角。圆角的视觉焦点向内收敛，给人一种"安全、可点击"的心理暗示；直角的视觉焦点向外发散，容易与页面边框混淆，降低可点击感知。

此外，圆角在物理世界中与"可按压的物体"（如键盘按键、遥控器按钮）高度关联，符合用户的心理模型。Material Design 将圆角半径定义为 4dp（密度无关像素），Ant Design 定义为 2px（小元素）到 8px（大元素），都是为了在"现代感"和"可点击感"之间取得平衡。

为什么禁用状态要降低透明度而不是变灰？

禁用状态的设计目标很明确：让用户一眼看出"这个元素现在不能交互"。降低透明度（opacity: 0.5）比单纯变灰更有效，因为透明度变化同时影响了元素与背景的对比度，使禁用元素在视觉上"退后"一层，与可用元素形成明确的前景-背景关系。如果只变灰（如从蓝色变成灰色），在复杂的页面上，灰色禁用按钮可能与页面的中性色元素混淆，辨识度不足。

原理

组件状态机的完整性

现代设计系统将每个交互组件视为一个有限状态机（Finite State Machine）。以按钮为例，一个完整的按钮必须定义以下状态及状态间的转换规则：

状态触发条件视觉表现设计目的
默认（Default）无交互标准颜色、标准阴影表明元素可交互
悬停（Hover）鼠标移入颜色变浅/变深、阴影增强确认交互目标、提供反馈
激活（Active）鼠标按下颜色加深、位移 1px模拟物理按压感
聚焦（Focus）键盘 Tab 选中外发光（outline）支持无障碍键盘导航
禁用（Disabled）条件不满足透明度 50%、无 hover 效果阻止交互、提示前置条件
加载（Loading）异步操作进行中转圈动画、禁用点击防止重复提交

缺少任何一个状态，都会导致用户体验的断裂。例如：没有 Focus 状态的按钮，键盘用户无法知道自己当前选中了哪个元素；没有 Loading 状态的提交按钮，用户可能因网络延迟而重复点击。

Ant Design 按钮类型的决策推导

Ant Design 将按钮分为五种类型，这个分类不是随意为之，而是基于操作重要性的严格排序：

  1. 主按钮（Primary）：一个页面上只有一到两个，用于最重要的操作（提交、保存）。使用品牌主色 + 实心填充，形成最强的视觉权重。
  2. 次按钮（Default / Ghost）：最常用的按钮，用于普通操作（取消、返回）。使用描边或浅色背景，视觉权重低于主按钮。
  3. 虚线按钮（Dashed）：用于添加、导入等次要操作。虚线边框暗示"创建一个容器"或"添加一项"，视觉上比次按钮更弱。
  4. 文字按钮（Text）：用于不重要的操作，或者空间有限的场景。去掉边框和背景，仅保留文字，视觉权重最低。
  5. 链接按钮（Link）：用于跳转外部页面或打开新窗口。样式与超链接一致，暗示"离开当前页面"。

这个序列的本质是视觉权重的梯度控制：页面上同时出现的按钮越多，越需要严格的权重分配，否则用户的注意力会被分散，不知道哪个是主要操作。

输入框的交互设计原理

输入框的设计目标是降低用户的输入成本，同时防止错误。以下是关键设计决策的推导：

  • 高度统一：标准 36px，大 44px，小 28px。与按钮高度对齐（如标准按钮 36px），方便在表单中并排布局，保持视觉基准线一致。
  • 聚焦状态要明显：边框变色 + 外发光（box-shadow）。根据格式塔的图形-背景关系，外发光创造了一个"光环"，将输入框从背景中分离出来，用户无需思考就能定位当前焦点。
  • placeholder 只是辅助：placeholder 的颜色对比度通常较低（符合辅助信息的层级），且用户输入后 placeholder 消失。如果 placeholder 承载了必要信息（如密码规则），用户输入后就看不到了，导致错误。因此，重要信息必须用 label 或 helper text 承载。

论证

按钮状态转换的完整性检验

一个合格的按钮组件必须满足状态覆盖性和状态互斥性两个条件：

  • 状态覆盖性：用户在任何场景下与按钮交互，都必须能感知到当前状态。如果用户点击按钮后没有任何反馈，他会怀疑自己是否"点中了"，进而重复点击。
  • 状态互斥性：同一时刻，按钮只能处于一个确定状态。例如，禁用状态和加载状态不能同时激活（逻辑上，加载意味着按钮正在工作，但禁用意味着按钮不可用）。

Ant Design 的按钮状态机遵循以下转换规则：Default → Hover → Active → Default（正常点击流程）；Default → Focus → Active/Default（键盘导航流程）；Default → Loading → Default/Success/Error（异步操作流程）；任何状态 → Disabled（条件变化流程）。这个状态机的完整性保证了用户在任何交互路径下都能获得明确的反馈。

圆角半径的数学选择

圆角半径不是"越圆越好"。Ant Design 的圆角系统遵循尺寸比例原则：元素越小，圆角半径越小；元素越大，圆角半径越大。具体来说：

  • 小元素（标签、徽章、小按钮）：圆角 2px ~ 4px。小圆角在有限的空间内提供"柔和感"，但不占用过多内容区域。
  • 中元素（标准按钮、输入框、卡片）：圆角 4px ~ 8px。这是最常见的尺寸，平衡了现代感和内容利用率。
  • 大元素（模态框、大卡片）：圆角 8px ~ 12px。大圆角让大容器不显笨重，同时与内部的小圆角元素形成层级对比。

如果使用统一的圆角（比如所有元素都用 8px），小标签会显得"过于圆润"，大模态框则会显得"圆角不够"。尺寸比例原则确保了不同层级的元素在视觉上既有统一感又有区分度。

要点

  1. 按钮类型按重要性递减：Primary → Ghost → Dashed → Text → Link。一个页面上 Primary 按钮不超过两个。
  2. 危险操作用红色：删除、清空、退出登录等不可逆操作，用 btn-danger 给用户心理预警。
  3. 标签（Tag）用颜色表达语义：蓝色=进行中，绿色=成功，黄色=警告，红色=失败。不要让用户去猜。
  4. 徽章（Badge）只用于数字提示：消息未读数、待处理任务数。不要用它来装饰。
  5. 禁用状态要一眼看出：降低透明度到 50%，去掉 hover 效果，鼠标变成 not-allowed。

实验

创建一个 HTML 文件，复制以下代码并在浏览器中打开。这个实验用纯 CSS 实现一个带 hover、active、disabled 状态的按钮组件，你可以直接观察每个状态的视觉反馈。

EXPERIMENT

实验代码（保存为 button-states.html）：

<!DOCTYPE html>
<html>
<head>
  <style>
    body {
      font-family: sans-serif;
      padding: 40px;
      background: #F5F7FA;
    }
    .btn {
      display: inline-flex;
      align-items: center;
      justify-content: center;
      height: 36px;
      padding: 0 20px;
      font-size: 14px;
      font-weight: 500;
      border: 1px solid transparent;
      border-radius: 6px;
      background: #1677FF;
      color: #fff;
      cursor: pointer;
      transition: all 0.15s ease;
      box-shadow: 0 2px 0 rgba(5, 25, 55, 0.08);
    }
    /* Hover：颜色变浅，阴影增强 */
    .btn:hover {
      background: #4096FF;
      box-shadow: 0 4px 12px rgba(22, 119, 255, 0.25);
    }
    /* Active：颜色加深，向下位移 1px */
    .btn:active {
      background: #0958D9;
      transform: translateY(1px);
      box-shadow: 0 1px 0 rgba(5, 25, 55, 0.08);
    }
    /* Focus：外发光，支持键盘导航 */
    .btn:focus-visible {
      outline: none;
      box-shadow: 0 0 0 3px rgba(22, 119, 255, 0.25), 0 2px 0 rgba(5, 25, 55, 0.08);
    }
    /* Disabled：透明度降低，鼠标变禁止 */
    .btn:disabled {
      opacity: 0.5;
      cursor: not-allowed;
      box-shadow: none;
    }
    .btn:disabled:hover {
      background: #1677FF;
      box-shadow: none;
    }
    /* 状态展示 */
    .state-group {
      margin-bottom: 24px;
    }
    .state-label {
      font-size: 12px;
      color: #666;
      margin-bottom: 8px;
    }
  </style>
</head>
<body>
  <div class="state-group">
    <div class="state-label">默认状态 — 用鼠标悬停观察 Hover</div>
    <button class="btn">提交</button>
  </div>
  <div class="state-group">
    <div class="state-label">禁用状态 — 点击无效，无 Hover 反馈</div>
    <button class="btn" disabled>提交</button>
  </div>
  <div class="state-group">
    <div class="state-label">键盘导航 — 按 Tab 键选中，观察 Focus 光环</div>
    <button class="btn">提交</button>
  </div>
</body>
</html>

实验任务： 1) 用鼠标悬停、点击默认按钮，观察颜色变化和位移反馈； 2) 尝试点击禁用按钮，确认无反馈； 3) 按 Tab 键切换焦点，观察 Focus 光环； 4) 将 border-radius: 6px 改为 0，体会圆角对"可点击感"的影响。

常见错误

❌ 按钮主次不分

页面上有三个主按钮（蓝色），用户不知道哪个是主要操作。应当只有一个 Primary，其余用 Default 或 Text。

❌ 输入框没有聚焦样式

用户点了输入框，但没有任何视觉反馈，不知道自己有没有选中。必须加边框变色和外发光。

❌ 标签颜色与语义不符

用红色标签表示"进行中"，绿色标签表示"已删除"。颜色有强烈的心理暗示，不能乱用。

组件组合原则：

  1. 对齐：所有输入框的左边缘对齐，标签右对齐或顶对齐，形成清晰的视觉轨道。
  2. 间距：标签与输入框间距 8px（亲密性），两个表单项间距 16px（模块分隔）。
  3. 按钮层级："创建"是主按钮（Primary），页面上只有一个；"取消"是次按钮（Ghost），视觉上弱于主按钮。
  4. 状态完整：输入框有 Focus 状态（边框变色 + 外发光），必填项用红色星号标记，helper text 提供输入规则。
  5. 错误预防：在"创建"按钮旁预留空间用于展示表单级错误提示（如"项目名称已存在"）。

练习

  1. 表单对齐：创建一个表单页面，包含用户名（输入框，36px）、密码（输入框，36px）、提交（主按钮，36px）、重置（次按钮，36px）。确保所有元素高度严格对齐，并用 8px 网格控制间距。
  2. 标签系统：为一个任务列表页面设计标签系统，包含"待处理""进行中""已完成""已取消"四种状态，每种状态用对应的颜色标签。确保颜色语义与状态含义一致。
  3. 禁用状态：给上面的表单加上禁用状态（提交按钮在输入为空时禁用）。测试禁用按钮与可用按钮的视觉差异是否能在 1 秒内被识别。
  4. 状态机实现：用纯 CSS 为一个按钮实现完整的六状态样式（Default、Hover、Active、Focus、Disabled、Loading）。Loading 状态用 CSS 动画实现一个转圈图标。
