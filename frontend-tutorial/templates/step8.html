Step 8 响应式设计

移动端适配、弹性布局、断点策略

进阶

场景

同一个页面，在 27 寸显示器、13 寸笔记本、iPad、手机上都要能正常显示。响应式设计不是"把 PC 页面缩小到手机上"，而是根据不同屏幕尺寸重新排列内容，保证每个设备上的体验都最佳。

原理

移动优先（Mobile First）

先设计手机端的布局，然后再往大屏扩展。原因：

  • 手机空间最紧张，迫使你去思考"什么是最重要的"。
  • 从小往大加内容，比从大往小删内容更自然。
  • CSS 的 min-width 媒体查询比 max-width 更符合逻辑（ progressively enhance 渐进增强）。

断点选择

Ant Design 的断点：

  • xs：< 576px（手机）
  • sm：576px ~ 768px（大屏手机 / 小平板）
  • md：768px ~ 992px（平板）
  • lg：992px ~ 1200px（小笔记本）
  • xl：> 1200px（桌面）

实际项目中，通常只需要 2-3 个断点：手机（<768px）、平板（768px-1200px）、桌面（>1200px）。

弹性布局 vs 固定布局

  • 弹性布局（Fluid）：宽度用百分比或 flex，适应不同屏幕。
  • 固定布局（Fixed）：宽度写死 px，只适配特定尺寸。
  • 混合布局：容器固定最大宽度，内部用弹性布局。这是目前最主流的做法。

要点

  1. 用 Flexbox 和 Grid 做布局：它们天生支持响应式，配合 flex-wrap 和 gap 可以轻松实现多列变单列。
  2. 图片要自适应：max-width: 100%; height: auto;，防止图片撑破容器。
  3. 触摸目标不小于 44×44px：按钮、链接在手机上要足够大，方便手指点击。Ant Design 的按钮最小高度就是 28px，但在移动端建议至少 36px。
  4. 隐藏非核心内容：手机端可以把侧边栏收起、把次要信息折叠，优先展示核心内容。
  5. 测试真实设备：Chrome 的开发者工具只能模拟尺寸，不能模拟真实的触摸、滚动性能。必须在真机上测试。

常见错误

❌ 用 px 定义所有宽度

一个 400px 宽的卡片在手机上会撑出屏幕。应当用 max-width 或百分比。

❌ 断点太多太细

定义了 7 个断点，每个都要单独写样式，维护困难。2-3 个断点足够覆盖 95% 的场景。

❌ 忽略横屏模式

手机横过来后，高度变小，原本竖屏适配的布局可能会出问题。要测试横屏。

完整代码

/* 基础样式（移动端优先） */
.container {
  width: 100%;
  padding-left: 16px;
  padding-right: 16px;
}

.card-grid {
  display: flex;
  flex-direction: column;
  gap: 16px;
}

/* 平板 */
@media (min-width: 768px) {
  .container {
    padding-left: 24px;
    padding-right: 24px;
  }
  .card-grid {
    flex-direction: row;
    flex-wrap: wrap;
  }
  .card-grid > * {
    flex: 1 1 calc(50% - 8px);
  }
}

/* 桌面 */
@media (min-width: 1200px) {
  .container {
    max-width: 1200px;
    margin-left: auto;
    margin-right: auto;
  }
  .card-grid > * {
    flex: 1 1 calc(33.333% - 11px);
  }
}

练习

练习 1：用 Flexbox 实现一个三列卡片布局，要求在手机上一列、平板两列、桌面三列。不借助任何 CSS 框架。

练习 2：打开你的项目，在 Chrome DevTools 的手机模拟模式下测试每个页面，记录布局崩坏的地方。

练习 3：为前端设计系统教程的侧边栏添加响应式逻辑：PC 显示侧边栏，手机隐藏侧边栏并添加一个汉堡菜单按钮。
