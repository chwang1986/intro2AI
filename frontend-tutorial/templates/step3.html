Step 3 字体系统

字体族、字号层级、字重、行高与排版节奏

核心基础

场景

一个页面上通常有多种文字：大标题、小标题、正文、按钮文字、提示文字、代码。如果每种都用不同的字号和字体，页面就会显得凌乱。字体系统的目标是用最少的变量，覆盖所有文字场景。

起源

字体设计的历史可以追溯到 15 世纪古腾堡发明活字印刷术。早期的衬线体（Serif）如 Garamond、Times New Roman，其笔画末端的装饰性"衬线"源于雕刻石碑时的起笔和收笔痕迹。这些衬线在纸质媒介上起到了引导视线的作用，帮助读者在长篇印刷品中保持行进的节奏。

20 世纪 50 年代，瑞士国际主义风格推动了无衬线体（Sans-serif）的普及。Helvetica 等字体摒弃了装饰性衬线，以简洁的几何造型适应现代商业传播的需求。进入数字时代，屏幕的像素密度远低于印刷纸张，衬线体的小装饰在低分辨率屏幕上会变成视觉噪点，无衬线体因此成为屏幕显示的首选。

中文字体的演进则更为复杂。从铅字印刷的宋体、黑体，到激光照排时代的汉仪、方正，再到屏幕时代的阿里巴巴普惠体、思源黑体，每一次技术变革都伴随着字体设计目标的转变——从"在纸上清晰呈现"到"在屏幕上快速渲染、在小字号下依然可读"。

思考

为什么中文需要更大的行高？

中文和英文的排版需求存在根本差异。英文是拼音文字，字形高度一致（x-height），单词之间有空格分隔，天然形成了呼吸节奏。中文是方块字，每个字占据相同的字框，连续排列时如果没有足够的行距，会形成密不透风的"文字墙"。

此外，中文的笔画复杂度差异极大："一"字只有一横，"曦"字有二十笔。在 14px 及以下的小字号中，复杂笔画的字容易糊成一团。适当增加行高（line-height），不仅给视线换行提供了休息空间，也让每行文字之间的轮廓差异更容易被识别。

经验法则：中文正文行高至少 1.5，英文正文 1.4 即可；中文长文本（如文章、文档）建议 1.75，英文长文本 1.6 足够。Ant Design 的 line-height-normal 为 1.5，line-height-relaxed 为 1.75，正是基于这一差异。

为什么字重分级比字号分级更重要？

在移动端和小屏幕上，字号的变化空间有限（12px ~ 16px 是安全区）。如果仅靠字号区分层级，小字号之间的差异（如 12px vs 14px）可能不足一个像素在视网膜屏上的物理尺寸差异，用户根本感知不到。而字重的变化（Regular 400 vs Medium 500 vs Bold 700）在任意字号下都清晰可见，是更可靠的层级工具。

原理

屏幕渲染机制：为什么无衬线体更清晰

屏幕显示文字时，操作系统会将矢量字体栅格化（Rasterization）为像素点阵。衬线体（如宋体）的细小装饰笔画（衬线）在栅格化时，宽度可能不足一个像素，被渲染成不规则的锯齿或虚影，变成视觉噪点。无衬线体（如黑体）的笔画粗细均匀，横平竖直，在栅格化时更容易与像素网格对齐，保持边缘清晰。

现代操作系统使用子像素渲染（Subpixel Rendering，如 ClearType）和灰度抗锯齿来改善字体清晰度，但这些技术对笔画简单的无衬线体效果更好。这也是 Ant Design、TDesign、Material Design 都选择无衬线体作为默认字体的根本原因。

阿里巴巴普惠体的设计优化

阿里巴巴普惠体不是简单地将一款现有字体改名，而是针对屏幕显示进行了系统性优化：

  • 字腔放大：中文字符的内部空间（字腔）被适度放大，即使在小字号下，"口、日、目"等结构的内部空白依然可见，避免糊成一团。
  • 中宫收紧：字符的中心区域（中宫）略微收紧，使字形在屏幕上更紧凑、更有精神，不会因为屏幕拉伸而显得松散。
  • 覆盖中文、英文、数字，风格统一：避免了中文用黑体、英文用 Arial、数字用 Times 的拼凑感。
  • 字重齐全：Light / Regular / Medium / Bold / Heavy 五级字重，覆盖从正文到标题的所有场景。
  • 100% 免费商用：无版权风险，适合企业级产品。

字号梯度的信息架构逻辑

Ant Design 使用 12px、14px、16px、20px、24px、30px、38px 的梯度。这个梯度不是随意设定的，而是基于信息层级理论：

  1. 小字号（12→14→16）：以 2px 递增，对应辅助信息、正文、强调正文的层级。小字号区间需要精细区分，因为用户在这些区域执行的是"仔细阅读"任务。
  2. 大字号（20→24→30→38）：以 4px 或 6px 递增，对应小节标题、区块标题、页面大标题。大字号区间需要明显跳跃，因为用户在这些区域执行的是"快速扫描"任务，差异必须一眼可见。

这种"小字号密集、大字号稀疏"的梯度分布，符合人眼中央凹视觉（Foveal Vision）的特性：中央视野对细节敏感，适合小字号的精细区分；周边视野对整体轮廓敏感，需要大字号有足够的尺寸差异才能被捕捉。

行高的生理学基础

行高（line-height）本质上控制的是行间距（Leading，源自铅字时代的铅条）。适当的行间距让视线的回归运动（Saccade）有明确的终点：行高太小，回归运动会越过目标行，落入下一行（称为"串行"）；行高太大，回归运动的路程变长，阅读效率下降。

研究表明，中文的最佳行高约为字号的 1.5 ~ 1.75 倍。低于 1.5 时，串行错误率显著上升；高于 1.75 时，阅读速度开始下降，因为视线需要移动更长的距离才能找到下一行的起点。

论证

字号层级与信息架构的关系

信息架构（Information Architecture）的核心问题是"如何让用户快速找到需要的信息"。字号层级是信息架构的视觉表达形式，两者之间存在严格的映射关系：

信息层级字号字重用户任务视觉特征
页面级38px / 30pxBold 700确认"我在哪里"最大、最粗，占据视觉中心
区块级24px / 20pxSemibold 600理解"这一区块讲什么"明显大于正文，形成锚点
模块级18px / 16pxMedium 500识别"这一模块的功能"与正文有适度区分
正文级16px / 14pxRegular 400阅读详细内容最舒适的阅读尺寸
辅助级14px / 12pxRegular 400获取补充信息缩小但保持清晰

这个映射关系的本质是注意力经济学：页面上最重要的信息（页面标题）投入最多的注意力资源（最大的字号 + 最粗的字重），次要信息逐级递减。如果层级混乱（比如正文和标题字号只差 2px），用户的大脑无法快速分配注意力，导致"找不到重点"。

中英文混排的挑战

中文字体的默认行高和英文字体的默认行高不同。当一段文字中同时出现中英文时，如果字体 fallback 机制处理不当，会出现行高"跳动"的现象——中文行占用的空间比英文行大，导致整段文字的垂直节奏被打乱。

解决方案是显式定义行高（如 line-height: 1.5），而不是依赖字体的默认行高（normal）。显式行高会强制所有字符（无论中文还是英文）在同一个基线网格上对齐，消除跳动。

要点

  1. 一套字体族搞定所有场景：'Alibaba PuHuiTi', 'PingFang SC', 'Microsoft YaHei', sans-serif。Alibaba PuHuiTi 优先，没有就回退到系统字体。
  2. 代码用等宽字体：'JetBrains Mono', 'SFMono-Regular', monospace。等宽字体让代码对齐，方便阅读。
  3. 字重三级就够了：Regular（400）用于正文，Medium（500）用于小标题和按钮，Bold（700）用于大标题和重要数字。
  4. 行高根据场景调整：标题用 1.25（紧凑），正文用 1.5（舒适），长文本用 1.75（宽松）。
  5. 不要用 font-size 小于 12px 的文字：低于 12px 在移动端几乎无法阅读，这是 Ant Design 的硬性规定。

常见错误

❌ 字体超过两种

标题用阿里巴巴普惠体，正文用宋体，代码用 Courier，数字用 DIN。字体种类越多，页面越不统一。除非有特殊品牌需求，否则全站只用一套字体族。

❌ 字号跳跃不合理

标题 24px，副标题 22px，正文 20px。三者太接近，没有形成明显层级。层级之间至少差 4px 以上。

❌ 正文行高太小

正文用 line-height: 1.2，文字挤在一起，读起来很喘。中文正文建议至少 1.5。

深入理解 CSS Grid 布局
作者：张三 · 2024年3月15日 · 阅读时间 8 分钟

CSS Grid 是现代网页布局最强大的工具之一。与 Flexbox 不同，Grid 是二维布局系统，可以同时控制行和列。

在本文中，我们将从基础概念出发，逐步深入到复杂的网格模板和区域命名。无论你是初学者还是有经验的开发者，都能从中获得新的见解。

"Grid 让复杂的页面布局变得像画表格一样简单。"

首先，让我们创建一个最简单的网格容器：

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

上面的代码将容器分为三列，每列等宽，间距 16px。1fr 表示占据可用空间的一等份。

:root {
  /* 字体族 */
  --font-sans: 'Alibaba PuHuiTi', 'PingFang SC', 'Microsoft YaHei', sans-serif;
  --font-mono: 'JetBrains Mono', 'SFMono-Regular', 'Menlo', monospace;

  /* 字号 */
  --font-size-xs: 12px;
  --font-size-sm: 14px;
  --font-size-md: 16px;
  --font-size-lg: 18px;
  --font-size-xl: 20px;
  --font-size-2xl: 24px;
  --font-size-3xl: 30px;
  --font-size-4xl: 38px;

  /* 字重 */
  --font-weight-regular: 400;
  --font-weight-medium: 500;
  --font-weight-semibold: 600;
  --font-weight-bold: 700;

  /* 行高 */
  --line-height-tight: 1.25;
  --line-height-normal: 1.5;
  --line-height-relaxed: 1.75;
}

/* 使用 */
.page-title {
  font-size: var(--font-size-3xl);
  font-weight: var(--font-weight-bold);
  line-height: var(--line-height-tight);
}

.body-text {
  font-size: var(--font-size-md);
  font-weight: var(--font-weight-regular);
  line-height: var(--line-height-normal);
}

练习

  1. 字号审计：打开你现在在做的项目，统计一共用了多少种不同的 font-size。如果超过 8 种，尝试合并到 5 种以内，并建立字号层级表。
  2. 字体回退测试：用阿里巴巴普惠体（CDN 引入）写一个标题 + 两段正文的页面，在浏览器开发者工具中禁用网络字体，观察系统回退字体（PingFang SC / Microsoft YaHei）的显示效果是否可接受。
  3. 行高对比：把一段长文本（超过 300 字）分别用 line-height 1.2、1.5、1.75 展示，邀请 3 个人阅读，记录他们主观舒适度评分（1-5 分）。
  4. 字重层级：找一个只有 Regular 和 Bold 两种字重的页面，尝试加入 Medium（500）作为中间层级，观察信息层级是否更清晰。
