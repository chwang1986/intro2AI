Step 2 色彩系统

设计令牌、主色、辅色、功能色、中性色阶

核心基础

场景

页面上的按钮、文字、背景、边框都需要颜色。如果没有统一规划，很容易出现"这个蓝和那个蓝不一样"的情况。色彩系统就是把所有颜色提前定义好，每个颜色都有名字和用途，开发时直接引用。

起源

人类对色彩的系统研究始于 1666 年，牛顿用三棱镜将白光分解为七色光谱，首次证明色彩是光的一种属性。19 世纪，歌德在《色彩论》中提出色彩具有心理效应，为色彩心理学奠定基础。20 世纪初，蒙塞尔颜色系统（Munsell Color System）将色彩描述为色相（Hue）、明度（Value）、彩度（Chroma）三个维度，这是现代色彩模型的雏形。

进入数字时代，显示设备使用RGB 加色模型（红绿蓝三原色光叠加），而印刷使用CMYK 减色模型（青品黄黑油墨叠加）。然而，RGB 对人类不直观——调大 R 值会让颜色偏红，但具体变成什么颜色很难预测。1980 年代，HSB/HSL 模型（色相、饱和度、亮度）被引入，它更接近人类描述颜色的自然方式（"偏蓝的、淡一点的、亮一点的"），成为设计师的首选工具。

2010 年代后，随着无障碍设计（Accessibility）的普及，WCAG 对比度标准成为色彩系统的硬性指标。今天的 Ant Design、TDesign 等系统，都是基于 HSB 模型生成色板，再用 WCAG 公式验证对比度是否达标。

思考

为什么 HSB 比 RGB 更适合设计？

假设你要把一个按钮的 hover 状态调"更亮一点"：

  • 在 RGB 中，你需要同时调整 R、G、B 三个值。比如 #1677FF（主色）调亮后变成 #4096FF，但你很难凭直觉猜出这个变化规律。
  • 在 HSB 中，你只需要调整 B（Brightness）一个值。H=213（蓝色相）、S=100%（饱和度不变）、B=100%（亮度提高），逻辑清晰直观。

HSB 的另一个优势是系统性生成色板。Ant Design 的算法从主色出发，保持色相 H 不变，按固定规律调整 S 和 B，就能生成 10 个梯度的色板。而 RGB 模式下，同样的操作需要复杂的色彩空间转换。

为什么对比度不能凭感觉？

人眼对不同颜色的敏感度不同。比如黄色在白色背景上很显眼，但蓝色在黑色背景上就很难看清。WCAG 的对比度公式考虑了人眼的亮度感知特性（相对亮度计算），给出了客观的数值标准：普通文本至少 4.5:1，大文本至少 3:1。这不是审美偏好，而是基于视障用户（低视力、色盲）的实际可读性测试得出的结论。

原理

Ant Design 色板生成的底层算法

Ant Design 不是凭感觉选颜色，而是基于 HSB 色彩模型的数学推导。从一个主色（如 #1677FF，HSB: 213, 100%, 100%）出发，算法通过以下逻辑生成 10 个梯度：

  1. 色相锁定：保持 H 不变（或微调 <5°），确保色板始终在同一个色系内，不会出现"这个蓝和那个蓝不一样"的情况。
  2. 饱和度曲线：低亮度时适当提高饱和度（避免颜色发灰），高亮度时降低饱和度（避免颜色刺眼）。
  3. 亮度等差：按非线性规律调整 B 值，确保视觉上每个梯度的"深浅跳跃感"一致。人眼对暗部的变化更敏感，所以暗色阶的 B 值间隔更小。

这样做的好处是：颜色之间有过渡，不会跳跃感太强；可以保证不同亮度下的对比度都足够；hover、active、disabled 状态有自然的深浅变化。

WCAG 对比度标准的数学推导

WCAG 2.1 的对比度公式不是简单的颜色差值，而是基于人眼的亮度感知非线性特性：

/* 相对亮度计算（sRGB 色彩空间） */
L = 0.2126 * R + 0.7152 * G + 0.0722 * B

/* 对比度 */
Contrast = (L1 + 0.05) / (L2 + 0.05)

公式中的系数 0.2126、0.7152、0.0722 对应人眼对红、绿、蓝三种颜色的敏感度。人眼对绿色最敏感（0.7152），对蓝色最不敏感（0.0722）。这意味着：即使两种颜色的 RGB 差值相同，如果一种偏绿、一种偏蓝，人眼感知的对比度也会不同。WCAG 公式用科学方法量化了这种感知差异。

色彩心理学（实用版）

色彩的心理效应不是文化偏见，而是有生理基础的。例如，蓝色光波长较短（450-495nm），穿透力弱，在视网膜上聚焦的位置更靠前，因此眼睛不需要频繁调节焦距，产生"平静"的感觉。红色光波长较长（620-750nm），穿透力强，容易触发警觉反应。以下是界面设计中的色彩语义：

  • 蓝色：科技、信任、冷静。适合做品牌主色。
  • 绿色：成功、通过、正常。用于正面反馈。
  • 黄色/橙色：警告、注意。用于需要用户留意的信息。
  • 红色：错误、危险、删除。用于负面反馈。
  • 灰色：中性、辅助、禁用。用于文字、边框、背景。

论证

为什么 HSB 比 RGB 更适合生成系统色板？

RGB 是设备依赖型色彩空间，描述的是"显示器需要发出多少红、绿、蓝光"，而非人类感知的颜色属性。HSB 是感知型色彩空间，直接映射人类描述颜色的自然语言。以下是两种模型调整颜色的对比：

操作目标RGB 方式HSB 方式
让颜色更亮同时增加 R、G、B，比例难以确定增加 B（亮度）一个值
让颜色更鲜艳拉大 RGB 的差值，结果不可预测增加 S（饱和度）一个值
生成同色系色板需要复杂的色彩空间转换固定 H，调整 S 和 B
保持品牌一致性难以约束颜色范围H 的浮动范围可控（<±5°）

中性色为什么用透明度而非固定灰值？

Ant Design 的文字色使用 rgba(0,0,0,0.88) 而非固定值如 #262626，这个设计决策有深层考虑：

  1. 适应性：在不同背景色上，固定灰值可能需要重新定义；而透明度会自动与背景混合，在浅色背景上显深、在深色背景上显浅。
  2. 层级表达：0.88（主文本）、0.65（次要）、0.45（辅助）、0.25（禁用）形成等比递减的感知层级，人眼对"透明度差异"的敏感度比对"灰阶差异"更一致。
  3. 暗色模式兼容：切换到暗色主题时，只需将底色从白变黑，文字透明度无需修改，即可自然适配。

要点

  1. 用 CSS 变量定义颜色：不要写死十六进制，用 var(--color-primary)，方便全局换肤。
  2. 每个颜色至少定义三个状态：默认、hover、active。比如主色要有 --color-primary、--color-primary-hover、--color-primary-active。
  3. 功能色必须成对出现：前景色 + 背景色。比如成功提示，文字用 --color-success，背景用 --color-success-bg。
  4. 中性色用透明度表示层级：主要文字 rgba(0,0,0,0.88)，次要 0.65，辅助 0.45，禁用 0.25。
  5. 背景分层：页面背景 #F5F7FA，卡片背景 #FFFFFF，代码背景 #F6F8FA，形成自然的层级。

常见错误

❌ 颜色对比度不足

浅灰文字放在白底上，或者蓝色按钮配白色文字但蓝色太浅。建议用工具检查对比度，正文对比度至少 4.5:1。

❌ 功能色乱用

用红色表示"查看详情"，绿色表示"删除"。红色和绿色在界面中有强烈的语义，不能随意互换。

❌ 忽略暗色文字在有色背景上的表现

在蓝色背景上直接放黑色文字，看起来脏。有色背景上应当用白色文字。

颜色使用分析：

  • 页面背景：--bg-page: #F5F7FA，浅灰蓝让白色卡片浮起来。
  • 输入框边框：--border-color: #D9D9D9，中性灰不抢注意力。
  • 错误提示：边框用 --color-error，文字用 --color-error，背景用 --color-error-bg——功能色必须成对出现。
  • 成功提示：文字用 --color-success，背景用 --color-success-bg。
  • 主按钮：--color-primary，页面上只有一个 Primary 按钮。
  • 次按钮：btn-ghost，描边样式，视觉上弱于主按钮。

关键原则：功能色只用于对应语义。红色只出现在错误场景，绿色只出现在成功场景，不能互换。

完整代码

核心 CSS 变量定义：

:root {
  /* 主色 */
  --color-primary: #1677FF;
  --color-primary-hover: #4096FF;
  --color-primary-active: #0958D9;
  --color-primary-light: #E6F4FF;

  /* 功能色 */
  --color-success: #52C41A;
  --color-success-bg: #F6FFED;
  --color-warning: #FAAD14;
  --color-warning-bg: #FFFBE6;
  --color-error: #F5222D;
  --color-error-bg: #FFF1F0;

  /* 中性色 */
  --gray-1: #FFFFFF;
  --gray-2: #FAFAFA;
  --gray-3: #F5F5F5;
  --gray-4: #F0F0F0;
  --gray-5: #D9D9D9;
  --gray-6: #BFBFBF;
  --gray-7: #8C8C8C;
  --gray-8: #595959;
  --gray-9: #434343;
  --gray-10: #262626;

  /* 背景 */
  --bg-page: #F5F7FA;
  --bg-card: #FFFFFF;
  --bg-code: #F6F8FA;

  /* 文字 */
  --text-primary: rgba(0, 0, 0, 0.88);
  --text-secondary: rgba(0, 0, 0, 0.65);
  --text-tertiary: rgba(0, 0, 0, 0.45);
  --text-disabled: rgba(0, 0, 0, 0.25);
}

练习

  1. 色板生成：打开 Ant Design 的色板生成工具，输入一个你喜欢的颜色（如 #E60027），观察生成的 10 个梯度。记录 H 值的变化范围，是否保持在 ±5° 以内。
  2. 颜色审计：在现有项目里搜索所有 # 开头的颜色值，统计有多少种不同的颜色。尝试用 CSS 变量替换重复的颜色，将颜色种类减少到 10 种以内。
  3. 对比度测试：找一个你项目中的按钮，用 WebAIM 对比度检查器测试按钮文字与背景的对比度。如果低于 4.5:1，调整颜色直到达标。
  4. 暗色模式实验：将项目中的文字色从固定灰值（如 #595959）改为 rgba(0,0,0,0.65)，然后在深色背景下观察文字是否自动变浅。

判断题

  1. RGB 色彩空间比 HSB 更适合人类直观理解和调整颜色。（ ）

选择题

  1. 以下哪种颜色组合最符合 WCAG 的对比度要求？
    • A. 浅灰底（#F5F5F5）+ 白色文字
    • B. 蓝色底（#1677FF）+ 白色文字
    • C. 黄色底（#FFFBE6）+ 浅黄文字
    • D. 白色底（#FFFFFF）+ 浅灰文字（#BFBFBF）
  2. 设计一个错误提示时，正确的颜色组合是？
    • A. 红色背景 + 黑色文字
    • B. 浅红背景（#FFF1F0）+ 红色文字（#F5222D）
    • C. 绿色背景 + 红色文字
    • D. 黄色背景 + 蓝色文字
