Step 4 间距与布局

8px 网格、留白法则、容器、响应式断点

核心基础

场景

页面上的元素之间需要空隙，太挤会压抑，太散会松散。间距系统就是给这些空隙定规矩，让页面看起来整齐又不死板。

起源

网格系统的历史可以追溯到古希腊和古埃及的建筑比例法则，但现代意义上的页面网格诞生于 20 世纪初的瑞士国际主义风格。设计师 Josef Müller-Brockmann 在 1981 年出版的《Grid Systems in Graphic Design》中，系统阐述了如何将页面划分为等宽的栏（Column）和统一的间距（Gutter），从而实现理性的排版秩序。

这套方法最初用于印刷品（杂志、海报），核心思想是：所有的设计元素都应当对齐到隐形的网格线上。1984 年，Apple 推出 Macintosh 电脑，引入了图形用户界面（GUI），屏幕设计开始借鉴印刷网格的概念。但由于屏幕尺寸不固定（不同于印刷品的固定纸张），响应式网格成为新的挑战。

2010 年代，随着移动设备的普及，8px 网格系统成为业界共识。Google Material Design（2014）、Ant Design（2015）、TDesign（2021）都将 8px 作为基础单位。这个选择并非偶然，而是基于数学、生理和工程的综合考量。

思考

为什么 8px 是基准，而不是 5px 或 10px？

选择 8px 而非其他数字，有三个层面的原因：

  1. 数学层面：8 是最小的"高度合成数"之一（1×8、2×4），可以被 1、2、4、8 整除。这意味着 8px 网格可以灵活地分割为 4px（半格）、2px（四分之一格），在需要精细调整时仍有规律可循。10px 只能被 1、2、5、10 整除，灵活性不足。
  2. 工程层面：主流屏幕分辨率（1920×1080、2560×1440、3840×2160）都能被 8 整除，不会出现半个像素的情况。而 5px 或 7px 在某些尺寸下会产生非整数像素，导致渲染模糊。
  3. 感知层面：人眼对 8px 的增量有自然的节奏感。从 8px 到 16px、24px、32px，每次增加都是一个"明显的 but 不突兀"的跳跃。5px 的增量太小，用户感知不到变化；10px 的增量又太大，中间缺乏过渡。

为什么留白不是浪费？

初学者常常觉得"页面太空"，试图用内容填满每一个角落。但认知心理学的研究表明，信息密度与理解效率呈倒 U 型关系：密度太低（太空）会让用户觉得内容单薄；密度太高（太满）则会让用户产生认知过载，无法有效处理信息。

留白的作用有三个：1) 分组——间距大小暗示元素之间的关系；2) 呼吸——给用户的视觉系统提供休息点；3) 强调——周围的空白让重要元素更突出。Apple 的产品页面是留白的极致运用，大面积空白让产品图和核心文案成为唯一的视觉焦点。

原理

8px 网格系统的底层逻辑

Ant Design 和 TDesign 都使用 8px 作为基础单位，但这个选择背后有更深层的逻辑：

  1. 可整除性：8 可以被 1、2、4、8 整除，意味着 8px 网格可以灵活地分割为 4px（半格）或 2px（四分之一格）。在需要精细调整时（如图标与文字的对齐），设计师仍然可以基于 8px 的倍数进行推导，而不会陷入"7px vs 8px"的随意选择。
  2. 屏幕像素对齐：主流屏幕分辨率（1920×1080、2560×1440、3840×2160）都能被 8 整除。使用 8px 的倍数，元素边缘会精确对齐到物理像素，避免亚像素渲染导致的模糊。
  3. 韦伯-费希纳定律：人眼对刺激的感知遵循对数规律，而非线性规律。8px、16px、24px、32px 的增量在感知上形成近似等比的节奏，每个阶梯都"刚好能被注意到"。

留白与认知负荷

留白不是"空着没东西"，而是主动的信息组织行为。认知心理学中的认知负荷理论（Cognitive Load Theory）指出，人脑的工作记忆容量有限（约 4±1 个组块）。当页面信息密度过高时，用户的认知负荷超载，无法有效处理信息。

留白通过三种机制降低认知负荷：

  • 分组：间距大小暗示元素之间的关系。相关内容间距小（亲密性），不同模块间距大（分隔）。TDesign 建议模块间距至少 24px，否则用户会误以为这些内容属于同一组。
  • 呼吸：连续的阅读或操作会让视觉系统疲劳。留白提供了"视觉休息点"，让用户在处理下一组信息前有机会重置注意力。
  • 强调：周围的空白让重要元素更突出。一个被大面积空白包围的按钮，比被其他元素挤压的按钮更容易被注意到。

容器宽度的生理学依据

容器宽度的选择不是凭感觉，而是基于人眼视野的生理限制。人眼的清晰视野（Foveal Vision）只有约 2° 的视角，对应阅读距离 50cm 时的 1.75cm 宽度。超出这个范围的内容需要通过眼动（Saccade）来扫视，增加认知负担。

Ant Design 的容器宽度推荐：

  • 阅读型页面（680px ~ 860px）：对应约 60~80 个汉字/行的长度。研究表明，这个长度下读者的回归眼动（Return Sweep）错误率最低。
  • 工具型页面（1200px ~ 1440px）：仪表盘需要并排展示多个数据模块，较宽的容器允许 2~4 列布局，同时保证每列内容不会过宽。
  • 全屏展示（100%）：数据大屏需要最大化利用屏幕空间，但内容区域应当用网格分块，避免单块内容过宽。

论证

8px 网格 vs 10px 网格：为什么 8 胜出？

假设我们要设计一个间距系统，覆盖从最小间隙到最大模块间距的所有场景。对比 8px 和 10px 两种基准：

场景8px 网格10px 网格
图标与文字的间隙4px（半格）或 8px5px（半格，非整数）或 10px（过大）
表单标签与输入框8px10px
卡片内边距16px 或 24px20px 或 30px
模块间距24px 或 32px30px 或 40px
页面级留白48px 或 64px50px 或 60px（节奏不均匀）

10px 网格的问题在于：半格是 5px，在许多屏幕分辨率下会导致亚像素渲染；生成的间距序列（10、20、30、40、50）在感知上过于均匀，缺乏"小步快跑 + 大步跨越"的节奏感。而 8px 网格生成的序列（4、8、12、16、24、32、48、64）既有精细调整的余地，又有明显的层级跳跃。

Flexbox 为什么优于 Float？

在 CSS 布局的历史中，Float 最初是为了实现文字环绕图片而设计的，后来被"借用"来做页面布局。但 Float 有先天缺陷：

  1. 高度塌陷：父元素无法自动包裹浮动的子元素，需要 clearfix hack。
  2. 间距控制困难：Float 布局中，元素间距需要通过 margin 控制，但最后一个元素的 margin 需要手动清除，否则布局会错位。
  3. 对齐能力弱：Float 只能实现左对齐或右对齐，无法实现垂直居中或等分分布。

Flexbox 是专门为布局设计的模块。gap 属性可以精确控制子元素间距，且不会影响容器外部；align-items 和 justify-content 提供了完整的对齐能力；弹性伸缩（flex: 1）让元素可以自动填充剩余空间。这些特性使 Flexbox 成为现代布局的首选工具。

要点

  1. 所有间距都是 4px 的倍数：最小单位 4px（--space-1），然后 8px、12px、16px、20px、24px、32px、40px、48px、64px。
  2. 模块间距 ≥ 24px：两个卡片之间、两个表单区块之间，至少留 24px，否则视觉上会粘在一起。
  3. 容器水平居中：设置 max-width 和 margin: 0 auto，保证在大屏上不会拉得太宽。
  4. 移动端减少内边距：PC 上容器左右内边距 24px，移动端减少到 16px，不浪费宝贵的屏幕空间。
  5. 网格用 Flexbox，不用 Float：Flex 的 gap 属性可以精确控制子元素间距，而且天然支持响应式。

常见错误

❌ 间距随意，没有规律

这个元素 margin 15px，那个 margin 18px，另一个 margin 22px。统一改成 16px、20px、24px，立刻整齐。

❌ 容器没有最大宽度

在 27 寸显示器上，段落文字从屏幕最左边延伸到最右边，阅读时需要大幅度摆头。设置 max-width: 860px 或 1200px。

❌ 移动端和 PC 用一样的内边距

手机上左右各留 24px，中间内容区域只剩很少空间。移动端应当减少到 16px 甚至 12px。

练习

  1. 间距审计：打开一个你做的页面，用浏览器的开发者工具检查所有 margin 和 padding 值，统计有多少个不是 4 的倍数。把它们改成最近的 4 的倍数，截图对比修改前后的视觉效果。
  2. 响应式卡片：用 Flexbox 实现一个三列等宽的卡片布局，卡片间距 16px，在大屏上三列，平板两列，手机一列。所有间距必须是 8px 的倍数。
  3. 容器对比：用 container-narrow（max-width: 860px）包裹一篇长文章，对比 full-width 和 narrow 的阅读体验。记录自己阅读时眼球移动的疲劳感差异。
  4. 8px 网格化：找一个现有页面，将其所有间距统一为 8px 网格（4、8、12、16、24、32）。观察页面是否变得更整齐，同时注意是否有"太挤"或"太散"的地方需要微调。
