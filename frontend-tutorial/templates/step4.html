Step 4 间距与布局

8px 网格、留白法则、容器、响应式断点

核心基础

场景

页面上的元素之间需要空隙，太挤会压抑，太散会松散。间距系统就是给这些空隙定规矩，让页面看起来整齐又不死板。

原理

8px 网格系统

Ant Design 和 TDesign 都使用 8px 作为基础单位。原因很实际：

  • 8 是偶数，一半（4px）、两倍（16px）、三倍（24px）都是整数，计算方便。
  • 主流屏幕分辨率都能被 8 整除，不会出现半个像素的情况。
  • 人眼对 8px 的增量有自然的节奏感，不会觉得跳。

留白的心理学

留白不是"空着没东西"，而是主动的组织行为。相关内容之间的间距小（亲密性），不同模块之间的间距大（分隔）。TDesign 建议模块间距至少 24px，否则用户会误以为这些内容属于同一组。

容器宽度的选择

太宽的容器（比如 100%）会让长文本难以阅读，因为视线移动距离太长。Ant Design 的推荐：

  • 阅读型页面（文章、文档）：最大宽度 680px ~ 860px。
  • 工具型页面（后台、仪表盘）：最大宽度 1200px ~ 1440px。
  • 全屏展示（数据大屏）：100%，但内容区域用网格分块。

要点

  1. 所有间距都是 4px 的倍数：最小单位 4px（--space-1），然后 8px、12px、16px、20px、24px、32px、40px、48px、64px。
  2. 模块间距 ≥ 24px：两个卡片之间、两个表单区块之间，至少留 24px，否则视觉上会粘在一起。
  3. 容器水平居中：设置 max-width 和 margin: 0 auto，保证在大屏上不会拉得太宽。
  4. 移动端减少内边距：PC 上容器左右内边距 24px，移动端减少到 16px，不浪费宝贵的屏幕空间。
  5. 网格用 Flexbox，不用 Float：Flex 的 gap 属性可以精确控制子元素间距，而且天然支持响应式。

常见错误

❌ 间距随意，没有规律

这个元素 margin 15px，那个 margin 18px，另一个 margin 22px。统一改成 16px、20px、24px，立刻整齐。

❌ 容器没有最大宽度

在 27 寸显示器上，段落文字从屏幕最左边延伸到最右边，阅读时需要大幅度摆头。设置 max-width: 860px 或 1200px。

❌ 移动端和 PC 用一样的内边距

手机上左右各留 24px，中间内容区域只剩很少空间。移动端应当减少到 16px 甚至 12px。

完整代码

/* 间距令牌 */
:root {
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 20px;
  --space-6: 24px;
  --space-8: 32px;
  --space-10: 40px;
  --space-12: 48px;
  --space-16: 64px;
}

/* 容器 */
.container {
  width: 100%;
  max-width: 1200px;
  margin-left: auto;
  margin-right: auto;
  padding-left: var(--space-6);
  padding-right: var(--space-6);
}

.container-narrow {
  max-width: 860px;
}

/* 响应式：移动端减少内边距 */
@media (max-width: 768px) {
  .container {
    padding-left: var(--space-4);
    padding-right: var(--space-4);
  }
}

/* Flex 网格 */
.row {
  display: flex;
  flex-wrap: wrap;
  gap: var(--space-4);
}

.col {
  flex: 1 0 0%;
}

.col-6 {
  flex: 0 0 50%;
  max-width: 50%;
}

练习

练习 1：打开一个你做的页面，用浏览器的开发者工具检查所有 margin 和 padding 值，统计有多少个不是 4 的倍数。把它们改成最近的 4 的倍数。

练习 2：用 Flexbox 实现一个三列等宽的卡片布局，卡片间距 16px，在大屏上三列，平板两列，手机一列。

练习 3：用 container-narrow（max-width: 860px）包裹一篇长文章，对比 full-width 和 narrow 的阅读体验。
