Step 7 导航与布局

信息架构、浏览模式、导航组件与页面结构

组件层

场景

用户打开一个网站，如何快速找到自己想要的内容？导航是信息架构的具象化表达。从电商网站的分类菜单到后台管理系统的侧边栏，从文档站点的目录树到社交应用的底部 Tab，导航设计决定了用户能否在复杂的信息空间中不迷路。

起源

导航的物理原型是图书馆的书架分类系统。杜威十进制分类法（1876 年）将书籍按主题编号排列，读者通过目录卡片找到书的位置。这个系统有三个关键特征：层级结构（大类→小类→具体书籍）、唯一路径（每本书只有一个编号）、可预测性（编号规则固定）。

互联网的早期导航借鉴了这一模型。1991 年，Tim Berners-Lee 创建的第一个网站使用的就是简单的文本链接列表。随着网站规模扩大，导航逐渐演化出多种形式：顶部导航栏、侧边菜单、面包屑、标签页、分页器……

2000 年后，信息架构（Information Architecture, IA）成为一门独立学科。Jesse James Garrett 在《用户体验要素》中提出，导航属于"结构层"，介于"范围层"（功能规格）和"框架层"（界面布局）之间。好的导航设计必须在用户需求与业务目标之间找到平衡。

思考

为什么 F 型浏览模式决定了导航布局？

Nielsen Norman Group 的眼动追踪研究发现，用户在浏览网页时呈现典型的 "F" 型模式：先水平扫视页面顶部，然后向下移动，再水平扫视一段，最后沿左侧垂直扫视。

这一发现对导航设计的启示：

  • 顶部导航最重要：用户第一眼就会看到页面顶部的内容，因此核心导航必须放在这里。
  • 左侧是次优位置：F 型的垂直扫视线沿左侧进行，所以侧边导航也能获得较高的注意力。
  • 右侧内容容易被忽略：除非用户明确寻找，否则右侧边栏的内容经常被"视觉盲区"跳过。

为什么导航层级不能太深？

认知心理学中的"神奇数字 7±2"（George Miller, 1956）指出，人脑短期记忆能同时处理的信息单元约为 5-9 个。如果导航层级超过 3 层，用户需要记住的路径就会超出工作记忆容量，导致"我在哪里？"的迷失感。

实际设计中的经验法则：

  • 顶部导航的同级项不超过 7 个。
  • 侧边导航的层级不超过 3 层。
  • 任何页面都能在 3 次点击内到达（3-Click Rule）。

原理

信息架构的底层逻辑

导航是信息架构的"用户可见部分"。信息架构的核心任务是将海量内容组织成用户能理解的心理模型。常见的组织方式有四种：

  • 层级结构（Hierarchical）：树状分类，从上往下越来越具体。适合内容有明确从属关系的场景，如电商平台（数码→手机→智能手机）。
  • 序列结构（Sequential）：线性流程，一步一步推进。适合向导、表单、教程等场景。
  • 矩阵结构（Matrix）：允许用户选择自己的路径。适合内容关联复杂、用户目标多样的场景，如知识库、文档站点。
  • 数据库模型（Database）：动态生成导航，基于元数据筛选。适合商品列表、搜索结果等海量数据场景。

导航组件的语义与适用场景

不同导航组件对应不同的信息层级和用户意图：

  • 顶部导航（Top Navigation）：对应"站点级"导航，展示最核心的几个频道。适合页面数量少（3-7 个）的场景。
  • 侧边导航（Side Navigation）：对应"模块级"导航，容纳更多菜单项，支持展开/收起。适合后台管理系统等复杂场景。
  • 面包屑（Breadcrumb）：对应"路径级"导航，展示当前页面在层级结构中的位置。适合深层级页面（超过 2 层）。
  • 标签页（Tabs）：对应"内容级"导航，在同一页面内切换不同视图。适合内容可分类但不需要独立页面的场景。
  • 锚点导航（Anchor）：对应"页面内"导航，在长页面中快速跳转。适合文档、详情页等单页内容丰富的场景。

Flexbox 与 Grid 的布局机制对比

现代 CSS 提供了两种强大的布局工具，理解它们的底层机制有助于做出正确选择：

  • Flexbox（弹性盒子）：一维布局系统，沿主轴（main axis）排列元素。底层机制是计算每个 flex item 的"剩余空间分配比例"。适合导航栏、按钮组、卡片列表等线性排列场景。
  • Grid（网格）：二维布局系统，同时控制行和列。底层机制是定义显式网格轨道（track），再将元素放置到指定单元格。适合仪表盘、整体页面骨架等需要行列同时控制的场景。

论证

导航深度与用户体验的量化关系

多项用户研究表明，导航层级与任务完成率呈负相关：

  • 层级为 1-2 时，任务完成率约为 95%。
  • 层级为 3 时，任务完成率下降至 85%。
  • 层级为 4 及以上时，任务完成率低于 70%，且用户满意度显著降低。

这并非偶然。每一层导航都增加了用户的"决策点"——用户需要阅读选项、理解含义、做出选择。根据 Hick's Law（席克定律），决策时间随选项数量呈对数增长。深层级导航意味着多次决策，累积的时间成本和心理压力会迅速上升。

顶部导航 vs 侧边导航的效率对比

在 1920×1080 的桌面屏幕上，顶部导航栏的可视选项通常不超过 7 个（受水平空间限制），而侧边导航可以容纳 15-20 个一级菜单（配合滚动）。但是，顶部导航的每个选项获得的视觉注意力远高于侧边导航的中间项。

Ant Design 的建议是：如果总页面数 ≤ 7，用顶部导航；如果 > 7，用侧边导航。混合使用（顶部放全局导航、侧边放模块导航）也是一种常见策略，但要注意避免"双重导航"带来的认知混乱。

要点

  1. 当前页面要高亮：用户需要知道自己"在哪里"。高亮样式要足够明显（品牌色背景或左边框），且不能与 hover 状态混淆。
  2. 导航链接要有 hover 状态：鼠标移上去变色或变背景，让用户知道这是可点击的。hover 反馈延迟不应超过 0.15 秒。
  3. 面包屑用于深层级页面：当页面嵌套超过两层（如 首页 / 商品 / 数码 / 手机），用面包屑提供返回路径。面包屑的最后一项应当是纯文本（不可点击），表示当前位置。
  4. 避免导航项过多：如果顶部导航放不下，优先考虑合并次要入口（放入"更多"下拉菜单），而非缩小字号或挤在一起。
  5. 移动端导航要可收起：手机屏幕宽度有限，侧边导航应当可以收起为图标模式或隐藏到抽屉（Drawer）中。汉堡菜单（☰）虽然争议颇多，但在空间受限时仍是有效方案。
  6. 保持一致性：同一产品内的导航模式应当统一。如果 A 页面用顶部导航，B 页面用侧边导航，用户会感到困惑。

实验任务：

  1. 在桌面端（宽度 > 992px）观察导航栏和卡片布局。注意导航是水平排列，卡片是三列。
  2. 将窗口缩放到 768px-992px 之间，观察卡片变为两列，导航仍然水平排列。
  3. 继续缩放到 < 768px，观察导航变为汉堡菜单，卡片变为单列。
  4. 点击汉堡按钮，观察下拉菜单的展开/收起。
  5. 修改代码，尝试用 Flexbox 替代 Grid 实现卡片布局（提示：用 flex-wrap: wrap 和 flex: 1 1 300px），对比两种方案的代码简洁度。

常见错误

❌ 导航高亮不明显

当前页面和别的页面看起来完全一样，用户不知道自己在哪里。高亮应当用品牌色背景、左边框或加粗文字，与其他项形成明显对比。

❌ 导航层级太深

用户需要点击 5 次才能找到目标页面。超过 3 层的导航会让用户迷失方向。应当考虑扁平化结构，或将深层内容放到搜索中。

❌ 移动端导航没有适配

PC 端的水平导航栏直接搬到手机上，导致菜单项被挤成一团或溢出屏幕。移动端应当使用汉堡菜单、底部 Tab 或折叠面板。

❌ 面包屑代替主导航

面包屑是辅助导航，不能替代主导航。如果用户需要通过面包屑才能到达主要页面，说明主导航设计有问题。

练习

练习 1：用 Flexbox 实现一个水平导航栏，包含 Logo（左侧）和 5 个菜单项（右侧）。菜单项在 hover 时背景变色，当前页面项有底部边框高亮。

练习 2：用 Grid 实现一个仪表盘布局：顶部是 Header，左侧是 200px 宽的 Sidebar，右侧是主内容区。主内容区内部是一个 2×2 的卡片网格。

练习 3：为一个文档站点设计导航结构：首页、教程（含 HTML、CSS、JavaScript 三个子项）、组件库、关于我们。选择合适的导航组件组合，并说明理由。

测验

QUIZ

1. 用户在浏览网页时最典型的眼动模式是什么？

  • A. Z 型模式
  • B. F 型模式
  • C. 圆形模式
  • D. 随机扫视

答案：B。Nielsen Norman Group 的研究发现，用户在浏览网页时呈现 F 型模式：先水平扫视顶部，再向下垂直扫视左侧。

2. 根据席克定律（Hick's Law），导航设计应当遵循什么原则？

  • A. 选项越多，用户决策越快
  • B. 决策时间随选项数量对数增长，因此应当控制选项数量
  • C. 导航层级越深，用户完成率越高
  • D. 所有导航项应当平铺展示，避免层级

答案：B。席克定律指出，决策时间随选项数量增加而延长。导航选项应当精简，层级应当控制在 3 层以内。

3. Flexbox 和 Grid 的主要区别是什么？

  • A. Flexbox 是二维布局，Grid 是一维布局
  • B. Flexbox 是一维布局（行或列），Grid 是二维布局（行和列同时控制）
  • C. Flexbox 只能用于导航，Grid 只能用于卡片
  • D. 两者没有区别，可以任意互换

答案：B。Flexbox 沿单一轴线排列元素，适合线性布局；Grid 同时控制行列，适合复杂页面骨架。
