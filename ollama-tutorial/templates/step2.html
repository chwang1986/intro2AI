场景场景引入

你已经了解了 Transformer 和 Token 的概念。现在你想在自己的 Mac mini 上跑一个大模型，写一个 Python 脚本来调用它。第一步不是写代码，而是让模型"住"进你的电脑里。

起源Ollama 的诞生：让大模型"本地化"变得简单

2022 年底，ChatGPT 的发布震撼了整个世界。人们突然发现，大语言模型不仅能聊天，还能写代码、翻译、总结文档。但问题是：如果你想在自己的电脑上运行类似的能力，门槛高得惊人。

当时的标准流程是：先去 Hugging Face 下载动辄几十 GB 的模型权重文件，然后安装 PyTorch、CUDA、各种依赖库，再写一堆加载和推理代码。对于没有深度学习背景的开发者来说，这几乎是一道不可逾越的门槛。

2023 年，Mike Rowe 和 Jeffrey Morgan 创立了 Ollama 项目。他们的目标很简单：让开发者用一行命令就能在本地运行大模型。Ollama 把模型管理、推理引擎和 API 服务器打包成一个简单的应用程序，彻底屏蔽了底层复杂性。

Ollama 的名字来源于西班牙语 "olla"（锅），寓意把各种模型像食材一样"炖"在一起，用户只需要打开锅盖就能享用。这个项目迅速在开发者社区中流行开来，成为本地运行开源大模型的首选工具。

思考为什么要在本地跑模型？

既然 OpenAI、Google 和 Anthropic 都提供了强大的云端 API，为什么还要费力在本地部署模型？这是一个关键的架构决策，需要从多个角度思考。

角度 1：隐私与数据安全

当你把敏感数据发送到云端 API 时，数据会经过互联网到达第三方服务器。对于医疗记录、法律文档、企业内部数据等场景，这是不可接受的。本地运行意味着数据永远不出你的电脑。

角度 2：成本

云端 API 按 Token 收费。对于个人开发者的少量调用，成本可能不高；但对于高并发的生产环境，每月的 API 账单可能高达数千甚至数万美元。本地运行只需一次性购买硬件，后续调用完全免费。

角度 3：延迟与可靠性

本地调用是局域网内的 HTTP 请求，延迟通常在毫秒级。而云端 API 需要经过互联网传输，延迟可能达到数百毫秒甚至数秒。此外，本地运行不依赖外部网络，即使断网也能工作。

角度 4：定制化

本地模型可以被微调（fine-tuning）以适应特定领域。你可以用自己的数据训练模型，让它成为你的"私有专家"。云端 API 通常不支持这种深度定制。

但本地运行也有代价：你需要自己的硬件（GPU 或高性能 CPU），模型的选择受限于你的显存大小，而且你需要自己负责模型的更新和维护。作为工程师，你的任务是根据场景权衡这些利弊。

原理核心原理

Ollama 是什么

Ollama 是一个开源工具，做三件事：

  1. 下载模型：从 ollama.com/library 拉取开源大模型（Llama、Qwen、DeepSeek 等）
  2. 运行模型：把模型加载到内存/GPU，提供推理服务
  3. 暴露 API：在本地启动一个 HTTP 服务器，让你用 Python/JavaScript/命令行调用

它的设计哲学是"一行命令跑模型"。你不需要懂 PyTorch、CUDA 或模型权重格式，Ollama 帮你封装好了。

安装 Ollama

macOS（包括你的 Mac mini）：

# 命令行安装
curl -fsSL https://ollama.com/install.sh | sh

# 或者用 Homebrew
brew install ollama

安装完成后，Ollama 会在后台运行一个服务，默认监听 localhost:11434。

验证安装：

ollama --version
# 输出类似: ollama version 0.5.0

拉取和运行模型

Ollama 把模型打包成类似 Docker 镜像的格式。你需要先"拉取"（下载）模型，然后才能运行。

# 拉取 Llama 3.2 3B（轻量级，适合普通电脑）
ollama pull llama3.2

# 拉取 Qwen 2.5 7B（中文表现优秀）
ollama pull qwen2.5:7b

# 拉取 DeepSeek Coder（代码专用）
ollama pull deepseek-coder:6.7b

模型文件默认存放在 ~/.ollama/models/，大小从几 GB 到几十 GB 不等。拉取时可以选择不同尺寸的变体（如 qwen2.5:1.5b、qwen2.5:7b、qwen2.5:14b），冒号后面的是参数规模和量化方式。

运行模型进入交互模式：

ollama run qwen2.5:7b
# 进入对话模式，可以直接输入问题
>>> 你好
你好！很高兴见到你...

REST API 结构

Ollama 在本地暴露了一组 REST API。你的 Python 程序就是通过这些接口与模型对话的。

基础地址：http://localhost:11434

端点 方法 作用
/api/tags GET 列出本地已安装的模型
/api/generate POST 单轮文本生成（给 prompt，返回 completion）
/api/chat POST 多轮对话（传入消息列表）
/api/embeddings POST 生成文本的向量嵌入（用于语义搜索）
/api/pull POST 拉取模型（和命令行 ollama pull 等价）

所有接口都返回 JSON。请求体也是 JSON，包含 model（模型名）和 prompt（输入文本）等字段。

论证本地部署 vs 云端 API：怎么选？

在实际项目中，选择本地部署还是云端 API 是一个重要的架构决策。让我们来系统对比。

维度本地部署（Ollama）云端 API（OpenAI 等）
数据隐私数据不出本地，最高隐私数据发送到第三方服务器
成本模式一次性硬件投入，调用免费按 Token 计费，高频调用成本高
延迟毫秒级（本地 HTTP）百毫秒级（依赖网络）
模型选择受限于本地硬件（显存/内存）厂商提供的模型，通常更强
可用性断网可用，但需要自己维护依赖厂商服务稳定性
定制能力可微调、可量化、完全可控仅能通过 prompt 调整
上手难度需要安装和配置注册账号获取 API Key 即可

决策框架：如果项目涉及敏感数据、需要高频调用、或要求离线可用，选择本地部署；如果追求最强模型能力、低频调用、或希望快速原型验证，选择云端 API。在很多实际场景中，两者可以结合使用——核心敏感任务用本地模型，通用任务用云端 API。

为什么 Ollama 是本地部署的"最优解"？

在 Ollama 出现之前，本地运行大模型的选择包括：直接使用 Hugging Face Transformers 库、vLLM、llama.cpp 等。这些工具各有优势，但都有一个共同点——配置复杂。Ollama 的价值不在于发明了新的推理引擎，而在于把复杂的事情变得简单。它用类似 Docker 的模型管理方式（pull/run），把推理优化、API 封装和跨平台支持全部集成在一起。对于"我想快速跑起来"的开发者来说，Ollama 是最高效的选择。

要点核心要点

  • Ollama = 模型下载器 + 推理引擎 + HTTP API 服务器
  • 默认监听 localhost:11434，安装后自动后台运行
  • 模型需要先 ollama pull 下载，再 ollama run 运行
  • 核心 API 是 /api/generate（单轮）和 /api/chat（多轮）
  • 模型名可以带版本标签（如 qwen2.5:7b），冒号后是规模和量化

实验安装并运行你的第一个本地模型

这个实验将引导你完成 Ollama 的完整安装流程，并验证模型是否正常工作。

实验 1：安装 Ollama

  1. 打开终端，运行安装命令：curl -fsSL https://ollama.com/install.sh | sh（macOS/Linux）或从官网下载安装包（Windows）。
  2. 安装完成后，运行 ollama --version 确认安装成功。
  3. 运行 ollama list，此时应该显示空列表（还没有下载任何模型）。

实验 2：拉取并运行模型

  1. 运行 ollama pull qwen2.5:1.5b 拉取一个轻量级模型（约 1GB，适合大多数电脑）。
  2. 观察下载进度，记录下载耗时和模型文件大小。
  3. 下载完成后，运行 ollama run qwen2.5:1.5b 进入交互模式。
  4. 输入 "你好"，观察模型的响应速度和回答质量。
  5. 输入 /bye 退出。

预期现象：轻量级模型的响应通常很快（1~3 秒），但回答质量可能不如大模型。1.5B 参数对于简单对话足够，但复杂推理任务可能会出错。

实验 3：验证 API 服务

  1. 确保 Ollama 在后台运行（安装后通常自动启动）。
  2. 在终端运行：curl http://localhost:11434/api/tags。
  3. 观察返回的 JSON，确认其中包含你刚下载的模型。
  4. 运行以下命令测试生成接口：
curl -X POST http://localhost:11434/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5:1.5b",
    "prompt": "用一句话解释什么是本地部署",
    "stream": false
  }'

预期现象：返回一个 JSON，其中 response 字段包含模型生成的文本。done 字段为 true 表示生成完成。

错误常见错误

错误 1：Ollama 服务没启动就调用 API

如果收到 Connection refused，先检查 Ollama 是否在运行：ollama list 或 curl http://localhost:11434。

错误 2：模型名写错

Ollama 对模型名大小写敏感。qwen2.5:7b 和 Qwen2.5:7b 不是同一个。用 ollama list 查看本地确切的模型名。

错误 3：磁盘空间不足

一个 7B 模型约 4~5GB，14B 约 9GB。拉取前确认磁盘空间，避免下载到一半失败。
