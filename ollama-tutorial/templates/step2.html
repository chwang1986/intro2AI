Step 2 — Ollama 部署与接口

本地安装、REST API 结构、模型管理

场景

你已经了解了 Transformer 和 Token 的概念。现在你想在自己的 Mac mini 上跑一个大模型，写一个 Python 脚本来调用它。第一步不是写代码，而是让模型"住"进你的电脑里。

原理

Ollama 是什么

Ollama 是一个开源工具，做三件事：

  1. 下载模型：从 ollama.com/library 拉取开源大模型（Llama、Qwen、DeepSeek 等）
  2. 运行模型：把模型加载到内存/GPU，提供推理服务
  3. 暴露 API：在本地启动一个 HTTP 服务器，让你用 Python/JavaScript/命令行调用

它的设计哲学是"一行命令跑模型"。你不需要懂 PyTorch、CUDA 或模型权重格式，Ollama 帮你封装好了。

安装 Ollama

macOS（包括你的 Mac mini）：

# 命令行安装
curl -fsSL https://ollama.com/install.sh | sh

# 或者用 Homebrew
brew install ollama

安装完成后，Ollama 会在后台运行一个服务，默认监听 localhost:11434。

验证安装：

ollama --version
# 输出类似: ollama version 0.5.0

拉取和运行模型

Ollama 把模型打包成类似 Docker 镜像的格式。你需要先"拉取"（下载）模型，然后才能运行。

# 拉取 Llama 3.2 3B（轻量级，适合普通电脑）
ollama pull llama3.2

# 拉取 Qwen 2.5 7B（中文表现优秀）
ollama pull qwen2.5:7b

# 拉取 DeepSeek Coder（代码专用）
ollama pull deepseek-coder:6.7b

模型文件默认存放在 ~/.ollama/models/，大小从几 GB 到几十 GB 不等。拉取时可以选择不同尺寸的变体（如 qwen2.5:1.5b、qwen2.5:7b、qwen2.5:14b），冒号后面的是参数规模和量化方式。

运行模型进入交互模式：

ollama run qwen2.5:7b
# 进入对话模式，可以直接输入问题
>>> 你好
你好！很高兴见到你...

REST API 结构

Ollama 在本地暴露了一组 REST API。你的 Python 程序就是通过这些接口与模型对话的。

基础地址：http://localhost:11434

端点 方法 作用
/api/tags GET 列出本地已安装的模型
/api/generate POST 单轮文本生成（给 prompt，返回 completion）
/api/chat POST 多轮对话（传入消息列表）
/api/embeddings POST 生成文本的向量嵌入（用于语义搜索）
/api/pull POST 拉取模型（和命令行 ollama pull 等价）

所有接口都返回 JSON。请求体也是 JSON，包含 model（模型名）和 prompt（输入文本）等字段。

要点

  • Ollama = 模型下载器 + 推理引擎 + HTTP API 服务器
  • 默认监听 localhost:11434，安装后自动后台运行
  • 模型需要先 ollama pull 下载，再 ollama run 运行
  • 核心 API 是 /api/generate（单轮）和 /api/chat（多轮）
  • 模型名可以带版本标签（如 qwen2.5:7b），冒号后是规模和量化

常见错误

错误 1：Ollama 服务没启动就调用 API

如果收到 Connection refused，先检查 Ollama 是否在运行：ollama list 或 curl http://localhost:11434。

错误 2：模型名写错

Ollama 对模型名大小写敏感。qwen2.5:7b 和 Qwen2.5:7b 不是同一个。用 ollama list 查看本地确切的模型名。

错误 3：磁盘空间不足

一个 7B 模型约 4~5GB，14B 约 9GB。拉取前确认磁盘空间，避免下载到一半失败。

示例：用 curl 测试 API

# 列出本地模型
curl http://localhost:11434/api/tags

# 单轮生成
curl -X POST http://localhost:11434/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5:7b",
    "prompt": "用一句话解释什么是 Token",
    "stream": false
  }'

# 响应结构
{
  "model": "qwen2.5:7b",
  "created_at": "2026-07-11T10:00:00Z",
  "response": "Token 是大语言模型处理文本时的最小单位...",
  "done": true
}

练习

  1. 在你的电脑上安装 Ollama，拉取一个轻量级模型（如 llama3.2 或 qwen2.5:1.5b），用命令行和它对话。
  2. 用 curl 调用 /api/tags，看看你本地有哪些模型。
  3. 尝试用 curl 调用 /api/generate 生成一段文字，观察响应的 JSON 结构。
